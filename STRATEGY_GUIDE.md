## 1. 戦略の全体像

### ひとことで言うと

> **「ブレイクアウト主力 + V字回復補助 を、相性の良い 6通貨で同時運用する」**

### 構成図

```
┌──────────────────────────────────────────────────────────┐
│              口座 (100万円 / 10,000 USD 想定)               │
│                                                            │
│  リスク 1%/トレード, 最大DD 20% で運用停止                   │
└──────────────────────────────────────────────────────────┘
           │                            │
           ▼                            ▼
   ┌──────────────────────┐    ┌─────────────────────────┐
   │  主力: TrendBreakV1  │    │  補助: H4 T5+MACD+BB    │
   │  (H1 ブレイクアウト)  │    │  (急落V字回復+停滞ブレイク)│
   │                      │    │                         │
   │  RR 1:3 固定         │    │  RR 1:2 (SLはV字安値)   │
   │  ATR×1.5 stop        │    │  実戦用フィルタON        │
   │                      │    │  ロング専用 (現時点)      │
   │  Trades/年: 38       │    │  Trades/年: 3-4         │
   └──────────────────────┘    └─────────────────────────┘
           │                            │
           └─────────────┬──────────────┘
                         ▼
         ┌────────────────────────────────────┐
         │ 6通貨ペア (H1+H4):                 │
         │ XAUUSD / USDJPY / EURJPY           │
         │ GBPJPY / CHFJPY / SILVER           │
         │ ※AUDJPY は除外                     │
         └────────────────────────────────────┘
```

### 同時運用の意味

- **同じ通貨ペアで両方シグナルが出たら → 後発を見送り** (`trendbreak_priority_add_t5_when_free` 採用案)
  - もしくは `all_trades`: 両方フル採用 (検証ではこちらが最高数字)
- **異なる通貨ペアでは独立にポジション可** (リスク分散)
- TrendBreakV1 は H1, H4 T5 は H4 で動作する → 時間軸も分散

## 2. 2つの戦略の中身

### 2-1. TrendBreakV1 HYBRID ([`pine/production/TrendBreakV1_Final.pine`](pine/production/TrendBreakV1_Final.pine))

#### 役割
20日〜数か月級の高安値更新を使い、伸びる相場を取りに行く**主力**手法。

#### エントリー条件 (Long の場合)
1. 現在価格が「過去 N本の高値」を上方ブレイク
2. 直近 K本は不接触ゾーン (押し目を作っていない)
3. セッションフィルタ (通貨ごとに有効時間帯のみ)
4. クールダウン中ではない
5. ATR が異常 (急騰急落直後) ではない

#### SL/TP
- SL = エントリー − ATR × 1.5
- TP = エントリー + (SL距離 × 3) → **リスクリワード 1:3**

#### 通貨別 HYBRID 最適化パラメータ (Auto preset で自動選択)

| 通貨 | lookback | exclude | RR | セッション | 期待Trade/年 |
|---|---|---|---|---|---|
| XAUUSD | 90 | 24 | 3.0 | NY only | 8 |
| SILVER | 60 | 16 | 3.0 | NY only | 6 |
| EURJPY | 80 | 20 | 2.5 | EU+NY | 6 |
| GBPJPY | 80 | 24 | 2.5 | EU+NY | 6 |
| USDJPY | 60 | 16 | 2.5 | EU+NY | 7 |
| CHFJPY | デフォルト | デフォルト | 2.5 | EU+NY | 7 |

### 2-2. H4 T5 + MACD + BB ([`pine/production/h4_t5_macd_bb_live_ready.pine`](pine/production/h4_t5_macd_bb_live_ready.pine))

#### 役割
急落後の **V字回復を環境認識として使い**、その後に高値停滞・再ブレイク・MACD・BB が一致する場面だけに入る**補助**手法。

#### エントリー条件 (Long のみ, 現時点)

1. **急落の確認**: pivot high → pivot low の急落 (ATR×3.5以上, 0.30 ATR/本以上の速度)
2. **V字回復候補**: 急落幅の **61.8%〜80.0%** まで回復している
3. **回復速度**: 急落本数の **1.20倍以内** で回復 (例: 下落10本 → 12本以内)
4. **MACD**: ヒストグラムが3本前より上昇
5. **BB位置**: **0.75〜1.00** (Strict プリセット)
6. **BB幅**: ATR×7 以下 (4超はロット半減)
7. **トリガー**: V候補後 **16本以内** に以下のいずれか:
   - **高値停滞ブレイク** (stagnation)
   - **押し後の戻り高値再ブレイク** (rebreak)
   - 両方同時 (stagnation+rebreak) ← 最強
8. **騙し回避**: BB位置 > 0.95 または MACD slope3 ≤ 0.03 の単独rebreak は見送り
9. **年末年始除外**: 12/15-1/10 ではない

#### SL/TP
- SL = V字の安値 − ATR × 0.25
- TP = エントリー + (SL距離 × 2) → **リスクリワード 1:2**
- 最大保有 = H4 で 180本 (=30日)

#### プリセット一覧

| プリセット | BB位置 | BB幅 | V速度 |
|---|---|---|---|
| Current 0.60-1.10 | 0.60〜1.10 | 上限なし | REC1.0〜1.5 |
| Robust 0.75-1.05 | 0.75〜1.05 | ≤7 | REC1.0〜1.5 |
| **Strict 0.75-1.00** ⭐ | **0.75〜1.00** | **≤7** | **REC1.0〜1.5** |
| Balanced REC1.2 (V速度) | - | - | **下落の1.20倍以内** |

**推奨**: `Strict 0.75-1.00 + width<=7` × `Balanced REC1.2` × 騙し回避フィルタON

## 4. TradingView へのデプロイ手順

### 必要なもの

- TradingView **Pro+ 以上** (Strategy のアラート機能のため)
- 推奨ブローカー: スプレッドが小さいFX業者 (Oanda, IG, Forex.com, OandaJP, 楽天証券 など)

### 手順

#### Step 1: チャートを 12枚開く (各通貨で H1 + H4)

| Tab | シンボル | TF | 戦略 |
|---|---|---|---|
| 1 | XAUUSD | H1 | TrendBreakV1 |
| 2 | XAUUSD | H4 | H4 T5 MACD BB |
| 3 | USDJPY | H1 | TrendBreakV1 |
| 4 | USDJPY | H4 | H4 T5 MACD BB |
| 5 | EURJPY | H1 | TrendBreakV1 |
| 6 | EURJPY | H4 | H4 T5 MACD BB |
| 7 | GBPJPY | H1 | TrendBreakV1 |
| 8 | GBPJPY | H4 | H4 T5 MACD BB |
| 9 | CHFJPY | H1 | TrendBreakV1 |
| 10 | CHFJPY | H4 | H4 T5 MACD BB |
| 11 | SILVER (XAGUSD) | H1 | TrendBreakV1 |
| 12 | SILVER (XAGUSD) | H4 | H4 T5 MACD BB |

> Pine v5 の制約上、1チャートに別TFのStrategyを2つ載せるのは推奨しません。
> 別チャートで管理する方が、検証結果との一致度が高くなります。

#### Step 2: H1チャートに TrendBreakV1 を貼る

1. Pine Editor を開く
2. [`pine/production/TrendBreakV1_Final.pine`](pine/production/TrendBreakV1_Final.pine) の中身をコピペ
3. 「Add to chart」
4. パラメータ:
   - `preset` = **`Auto`** (通貨を自動判別)
   - `riskPct` = **`1.0`** (1%)
   - `maxDDPctIn` = **`20`** (DD 20%で停止)

#### Step 3: H4チャートに H4 T5 MACD BB を貼る

1. Pine Editor を開く
2. [`pine/production/h4_t5_macd_bb_live_ready.pine`](pine/production/h4_t5_macd_bb_live_ready.pine) の中身をコピペ
3. 「Add to chart」
4. パラメータ:
   - `analysisTf` = **`240`** (H4)
   - `requireTfMatch` = **ON** (推奨)
   - `filterPreset` = **`Strict 0.75-1.00 + width<=7`**
   - `vShapePreset` = **`Balanced REC1.2`**
   - `useGuardFilters` = **ON**
   - `skipHoliday` = **ON**

#### Step 4: アラート設定

各 Strategy で:
- Right click → Add alert
- Condition: Strategy 名
- Options: **"Once Per Bar Close"**
- 通知: Webhook/Email/Mobile App

H4 T5 MACD BB は `Any alert() function call` を選ぶと、Signal/Entry/SL/TP/Exit を全て通知できます。

## 5. リスク管理ルール

### 黄金律

1. **1トレード 1% リスクを超えない**
2. **同時保有 6ポジション (=6%まで)** を超えない
3. **DD 20% で運用即停止**
4. **連敗 13回 → ロット半減 (任意ルール)**
5. **H4 T5 MACD BB は最初 0.25R-0.5R から開始**、30件後に通常リスク
6. **年初に資金リセット** (毎年 1月1日にロット計算をやり直す)

### ロット計算 (例)

```
口座 1,000,000 JPY
リスク 1% = 10,000 JPY
ATR×1.5 = 15 pips (例)
USDJPY: 1 lot = 1 pip = 100,000 JPY × 0.01 = 1,000 JPY

ロット = 10,000 JPY / 15 pips / 1,000 JPY/pip/lot = 0.67 lot
```

> Pine Script の `calcQty()` で自動計算される。手動で発注する場合のみ意識。

### 重要: 「同時最大リスク」のチェック

- 6通貨×2戦略 = 12個のシグナルが理論上同時発火する可能性
- だが実際は、相関のある通貨 (XAUUSD と SILVER、JPYクロス全般) は同方向に動きがち
- **同方向で 3ポジション以上**になりそうなら、新規エントリーを 1つスキップする
- これは Pine では制御できないので、**手動で管理する**

## 7. 運用中の判断基準

### 日次チェック

- [ ] チャート12枚が正しく表示されているか
- [ ] アラートが機能しているか (テスト発火推奨)
- [ ] スプレッドが拡大していないか (重要指標発表時など)

### 週次チェック

- [ ] 週ごとの実 R が想定範囲 (±5R) か
- [ ] 通貨別の成績バランス
- [ ] H4 T5 のシグナル発火数 (月0-1件が想定)

### 月次チェック

- [ ] 月別 R 集計が想定範囲内 (-3R 〜 +10R/月)
- [ ] DD が 10R を超えていないか
- [ ] 連敗回数が 10回を超えていないか

### 異常時の対応

| 症状 | 対応 |
|---|---|
| 連敗 7回 | ロット維持で継続観察 |
| 連敗 10回 | 一旦 1週間休む |
| 連敗 13回 | ロット半減 → 5勝で復旧 |
| DD 10R | ロット 2/3 → DD回復まで |
| DD 15R | ロット 1/3 → DD回復まで |
| **DD 20R** | **運用停止** → 環境分析・パラメータ再検証 |
| 1月で -10R | 戦略の前提が崩れている可能性 → 検証やり直し |

### 環境変化への対応

| 環境 | TrendBreakV1 | H4 T5 MACD BB | 全体方針 |
|---|---|---|---|
| 強いトレンド相場 | 🟢 強い | (発火しにくい) | TBに任せる |
| レンジ相場 | ⚠️ 弱い | ⚠️ 発火少 | 両方休む |
| 高ボラ (急落後) | 🟢 強い | 🟢🟢 大化け | T5 のチャンス |
| 低ボラ (VIX<15) | ⚠️ 苦戦 | ⚠️ 発火少 | 2018年型 |
| ニュース直後 | ATR異常で除外 | V字発生? | 様子見 |

## 8. トラブルシューティング

### Pine Script でエラーが出る

- TradingView 上で「Pine v5 用」と表示されているか確認
- `max_lines_count=500` などの設定でエラー → TV のプランによっては数値を下げる
- 行数が500を超えるためチャート登録できない場合 → **Pine スクリプトを再保存後**、再度Add

### バックテストの数字が再現しない

- データ提供元が違う (Oanda vs Dukascopy vs FXCM)
- スプレッドの設定 (Pine の `slippage` を確認)
- セッション時間 (UTC vs JST) のズレ
- バックテストCSVとPine内蔵バックテストで数件のズレは想定範囲

### 思ったほどシグナルが出ない

- ✅ 正しい挙動。
  - TrendBreakV1: 年38トレード = 月3-4回
  - H4 T5 MACD BB: **年3-4トレード = 3-4ヶ月に1回** ⚠️
- もし「全くシグナル出ない」なら preset = Auto が効いてないかも → 手動で通貨選択

### 同時に同じ通貨で 2ポジション持ってしまう

- TradingView の Strategy は別 ID で発注するため別ポジになる
- → **H1 と H4 で別ポジ持つのはOK** (時間軸が違うため)
- 同じTFで2ポジは避ける → Pyramiding 設定を 0 (デフォルト)

### H4 T5 MACD BB が「シグナル出てるのに発注されない」

- 騙し回避フィルタで除外されている可能性
- BB位置 > 0.95 → 見送り
- V候補から 16本超 → 見送り

- ステータステーブルで確認可能

## 9. なぜこの戦略にしたか (経緯)

### 検証フェーズ全体図

```
[2025年]  Sai H1 オリジナル → PF 1.05 (弱い)
   ↓
[初期] TrendBreakV1 発見 → PF 1.79 / +194R (採用決定)
   ↓
[Sai 分解] 「急な揺り戻し+高値停滞」だけ抽出 → PF 1.47
   ↓
[ensemble v1] TrendBreakV1 + Sai Best → +278R / Calmar 17
   ↓
[ensemble v1 通貨フィルタ] TOP5構成 → Calmar 20.76
   ↓
[V字単独研究] H4 急落V字回復だけでは弱い (-7R) → 単独NG
   ↓
[T5フィルタ研究] V候補 + 高値停滞/再ブレイク + MACD + BB → PF 3.43 復活
   ↓
[実戦用フィルタ確立] BB<=0.95 + 16本以内 + 弱いrebreak除外 → PF 3.43, WR 60%
   ↓
[OOS 2025-2026] 15trades, +7.26R, PF 2.44 → 過剰最適化なし
   ↓
[ensemble v2] TrendBreakV1 + H4 T5 MACD BB → +219.9R / DD 11.9R ⭐
   ↓
[現状]  v2 を採用、Sai Best は archive ✅
```

### v1 (Sai Best) と v2 (H4 T5 MACD BB) の比較

| 指標 | v1 (Sai Best) | v2 (H4 T5 MACD BB) |
|---|---|---|
| Trades/10年 | 359 | 30 |
| Total R | +99R | +25R |
| PF | 1.50 | **3.43** |
| WR | 44.6% | **60.0%** |
| DD | 17.2R | **4.4R** |
| OOS検証 | なし | **+7.26R 維持** |
| 結論 | 件数◎ 品質△ | **件数△ 品質◎** |

→ v2 は **件数が少ないが品質が圧倒的に高い**。フォワード検証もパスしている。
→ TrendBreakV1 と組み合わせる時、件数より品質が活きる。

### 設計思想

1. **シンプルさ優先**: パラメータは少なく、ロジックは説明できる範囲に
2. **オーバーフィット警戒**: 通貨ごとに別パラメータは持つが、グリッドサーチでなく経済的合理性で決定
3. **コスト考慮**: バックテストはスプレッド+滑り込みで再評価済み
4. **OOS検証必須**: 2025-2026のデータで再現性確認
5. **心理的耐久性**: 連敗13回までは「想定内」。それを越える戦略は採用しない
6. **複数戦略の相補性**: 件数多×低品質より、件数少×高品質を補助に
