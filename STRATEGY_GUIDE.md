## 1. 戦略の全体像

### ひとことで言うと

> **「相補的な 2つのブレイクアウト戦略を、相性の良い 5通貨で同時運用する」**

### 構成図

```
┌────────────────────────────────────────────────────────┐
│                  口座 (10,000 USD 想定)                 │
│                                                         │
│  リスク 1%/トレード, 最大DD 20%で運用停止                  │
└────────────────────────────────────────────────────────┘
           │                            │
           ▼                            ▼
   ┌──────────────────┐         ┌────────────────────┐
   │  TrendBreakV1    │         │   Sai Best Method  │
   │  (ブレイクアウト) │         │   (V字+停滞ブレイク) │
   │                  │         │                    │
   │  RR 1:3 固定     │         │  RR 1:2 固定        │
   │  ATR×1.5 stop    │         │  ATR×1.5 stop       │
   └──────────────────┘         └────────────────────┘
           │                            │
           └─────────────┬──────────────┘
                         ▼
         ┌───────────────────────────────┐
         │ 5通貨ペア (H1):                │
         │ XAUUSD / SILVER / EURJPY      │
         │ GBPJPY / XAGUSD               │
         │ ※同一通貨内は先発優先         │
         └───────────────────────────────┘
```

### 同時運用の意味

- **同じ通貨ペアで両方シグナルが出たら → 先発を優先**
- **異なる通貨ペアでは独立にポジション可** (リスク分散)
- 重複却下は実測で **5.5% のみ** → ほぼ独立して機能

## 2. 2つの戦略の中身

### 2-1. TrendBreakV1 (`pine/TrendBreakV1_Final.pine`)

**エントリー条件 (Long の場合):**

1. 現在価格が「過去 N本の高値」を上方ブレイク
2. 直近 K本は不接触ゾーン (押し目を作っていない)
3. セッションフィルタ (通貨ごとに有効時間帯のみ)
4. クールダウン中ではない
5. ATR が異常 (急騰急落直後) ではない

**SL/TP:**
- SL = エントリー − ATR × 1.5
- TP = エントリー + (SL距離 × 3) → **リスクリワード 1:3**

**通貨別 HYBRID 最適化パラメータ** (Auto preset でこれが選ばれる):

| 通貨 | lookback | exclude | RR | セッション | 期待Trade/年 |
|---|---|---|---|---|---|
| XAUUSD | 90 | 24 | 3.0 | NY only | 14 |
| SILVER | 60 | 16 | 3.0 | NY only | 4 |
| EURJPY | 80 | 20 | 2.5 | EU+NY | 8 |
| GBPJPY | 80 | 24 | 2.5 | EU+NY | 7 |
| USDJPY | 60 | 16 | 2.5 | EU+NY | 5 |
| その他 | デフォルト | デフォルト | デフォルト | デフォルト | - |

### 2-2. Sai Best Method (`pine/sai_best_method_strategy.pine`)

**「急な揺り戻し+高値停滞」 = V字パターン + 停滞ブレイク**

**エントリー条件 (Long の場合):**

1. **中期トレンド一致**: 過去520本のSMA傾きが上向き
2. **短期トレンド一致**: 過去72本のSMA傾きが上向き
3. **モメンタム**: 過去12本で陽線が50%以上 + 値幅 > ATR×1.0
4. **停滞ゾーン形成**: 過去8本の高安レンジが ATR×1.2 以下
5. **ブレイクアウト**: 現在足が「停滞高値 + ATR×0.10」を超える
6. **V字パターン**: 過去96本の安値→現在の戻し率 ≥ 80%, 値幅 ≥ ATR×3.0
7. **節目から離れている**: 過去720本の最高値の ATR×1.25 以内ではない
8. **年末年始除外**: 12/15-1/10 ではない

**SL/TP:**
- SL = エントリー − ATR × 1.5
- TP = エントリー + (SL距離 × 2) → **リスクリワード 1:2**

**特徴:**
- TrendBreakV1 より「押し目→反発」を捉える
- WR がやや高い (44%) が RR は低い (1:2)
- 結果として TrendBreakV1 と異なる相場で稼ぐ → 相補性◎

## 4. TradingView へのデプロイ手順

### 必要なもの

- TradingView Pro 以上 (Strategy のアラート機能のため)
- 推奨ブローカー: スプレッドが小さいFX業者 (Oanda, IG, Forex.com など)

### 手順

#### Step 1: チャートを 5枚開く

| Tab | シンボル | TF |
|---|---|---|
| 1 | OANDA:XAUUSD | 1H |
| 2 | OANDA:XAGUSD | 1H |
| 3 | OANDA:EURJPY | 1H |
| 4 | OANDA:GBPJPY | 1H |
| 5 | OANDA:USDJPY (SILVER 代替) | 1H |

> SILVER専用シンボルが取れない場合は XAGUSD だけでもOK。

#### Step 2: 各チャートに Strategy を貼る

1. Pine Editor を開く
2. `pine/TrendBreakV1_Final.pine` の中身をコピペ
3. 「Add to chart」
4. 同じチャートに、もう1つ Strategy を追加:
5. Pine Editor を開く
6. `pine/sai_best_method_strategy.pine` の中身をコピペ
7. 「Add to chart」

#### Step 3: パラメータ確認

**TrendBreakV1:**
- `preset` = `Auto` のまま (通貨を自動判別)
- `riskPct` = `1.0` (1%)
- `maxDDPctIn` = `20` (DD 20%で停止)

**Sai Best Method:**
- `allowLong` = ✅
- `allowShort` = ✅ (リスク高ければ Long のみでも可)
- `riskPct` = `1.0` (1%)

#### Step 4: アラート設定

各 Strategy で:
- Right click → Add alert
- Condition: Strategy 名
- Options: "Once Per Bar Close"
- 通知: Webhook/Email/Mobile App

## 5. リスク管理ルール

### 黄金律

1. **1トレード 1% リスクを超えない**
2. **同時保有 5ポジション (=5%まで)** を超えない
3. **DD 20% で運用即停止**
4. **連敗 10回 → ロット半減 (任意ルール)**
5. **年初に資金リセット** (毎年 1月1日にロット計算をやり直す)

### ロット計算 (例)

```
口座 10,000 USD
リスク 1% = 100 USD
ATR×1.5 = 15 pips (例)
1 lot = 1 pip = 1 USD (USDJPYの場合)

ロット = 100 USD / 15 pips = 6.67 → 0.06 lot
```

> Pine Script の `calcQty()` で自動計算される。手動で発注する場合のみ意識。

## 7. 運用中の判断基準

### 日次チェック

- [ ] チャートが正しく表示されているか
- [ ] アラートが機能しているか (テスト alarms 推奨)
- [ ] スプレッドが拡大していないか (重要指標発表時など)

### 月次チェック

- [ ] 月別 R 集計が想定範囲内 (±5R/月) か
- [ ] DD が 10R を超えていないか
- [ ] 連敗回数が 7回を超えていないか

### 異常時の対応

| 症状 | 対応 |
|---|---|
| 連敗 7回 | ロット維持で継続観察 |
| 連敗 10回 | ロット半減 → 5勝で復旧 |
| DD 15R | ロット 1/3 → DD回復まで |
| DD 20R | **運用停止** → 環境分析 |
| 1月で -10R | 戦略の前提が崩れている可能性 → 検証やり直し |

### 環境変化への対応

| 環境 | TrendBreakV1 | Sai Best | コメント |
|---|---|---|---|
| トレンド相場 | 🟢 強い | 🟢 強い | 両方稼ぐ |
| レンジ相場 | ⚠️ 弱い | ⚠️ 弱い | 両方休む (filter で発火しない) |
| 高ボラ (VIX>30) | 🟢 強い | 🟢 強い | ATR比例で自動調整 |
| 低ボラ (VIX<15) | ⚠️ 苦戦 | ⚠️ 苦戦 | 2018年型 |

## 8. トラブルシューティング

### Pine Script でエラーが出る

- TradingView 上で「Pine v5 用」と表示されているか確認
- `max_lines_count=500` などの設定でエラー → TV のプランによっては数値を下げる

### バックテストの数字が再現しない

- データ提供元が違う (Oanda vs Dukascopy vs FXCM)
- スプレッドの設定 (Pine の `slippage` を確認)
- セッション時間 (UTC vs JST) のズレ

### 思ったほどシグナルが出ない

- ✅ 正しい挙動。年44トレード=月3-4回が想定値
- もし「全くシグナル出ない」なら preset = Auto が効いてないかも → 手動で通貨選択

### 同時に同じ通貨で 2ポジション持ってしまう

- TradingView の Strategy は別 ID で発注するため別ポジになる
- → **同じチャートで Strategy を2つ使うときは、Pyramiding 設定を 0** にする (デフォルト)

## 9. なぜこの戦略にしたか (経緯)

### 検証フェーズ

```
1. Sai H1 (オリジナル) 検証
   → PF 1.05 / +99R (10年) … 弱い
   → ただし「急な揺り戻し」だけは PF 1.47 / +99R … 抽出する価値あり

2. TrendBreakV1 検証
   → PF 1.56 / +551R (4モード combined)
   → conservative モード単独でも PF 1.74 / +184R
   → こちらをメインに採用

3. TrendBreakV1 精密監査 (audit)
   → スプレッド + 滑り考慮で 184R → 146R (-21%)
   → それでも PF 1.5 程度を維持 → 実運用OK

4. パラメータ緩和スタディ (relaxation)
   → 通貨別に最適パラメータが違うことを発見
   → HYBRID 最適化で頻度 +27%, R +23%

5. アンサンブル検証 (ensemble)
   → TrendBreakV1 + Sai Best Method
   → 重複却下わずか 5.5% → ほぼ独立
   → Calmar 12.3 → 17.0 に改善

6. 通貨フィルタ最適化 (portfolio_comparison)
   → CHFJPY/AUDJPY が「足を引っ張る」と判明
   → 除外で Calmar 17.0 → 20.8 → 採用
```

### 設計思想

1. **シンプルさ優先**: パラメータは少なく、ロジックは説明できる範囲に
2. **オーバーフィット警戒**: 通貨ごとに別パラメータは持つが、グリッドサーチでなく経済的合理性で決定
3. **コスト考慮**: バックテストはスプレッド+滑り込みで再評価済み
4. **心理的耐久性**: 連敗10回までは「想定内」。それを越える戦略は採用しない
5. **複数戦略の相補性**: 1戦略で +100R より、相補的な 2戦略で +200R を狙う

## 10. 今後の改良候補

### 短期 (1-3ヶ月)

- [ ] **2018年問題の解決**: 低ボラフィルタ追加 (ATR<過去平均なら見送り)
- [ ] **MTF確認の自動化**: H4 トレンド条件を Strategy に組み込む
- [ ] **時間帯フィルタの微調整**: JST 早朝のフェイクブレイクを除外
- [ ] **Webhook 経由の自動発注**: 3Commas や MT5連携

### 中期 (3-6ヶ月)

- [ ] **3つ目の戦略追加**: レンジ相場で稼ぐ戦略を加えて相補性さらに向上
- [ ] **OANDA API での実運用テスト**: デモ口座で3ヶ月
- [ ] **損切り改善**: 固定 ATR×1.5 ではなく構造的なSL (直近スイング安値) に変更

### 長期 (6ヶ月+)

- [ ] **機械学習による発火フィルタ**: 過去シグナルの勝敗パターンから低品質シグナルを除外
- [ ] **複数時間軸戦略**: H1 だけでなく H4, D1 でも検証して、時間軸分散
- [ ] **ニュースイベント連動**: ボラピーク前 30分は発火停止

### 絶対やらないこと

- ❌ **過度なパラメータ最適化** (オーバーフィット必至)
- ❌ **検証していないパラメータでの実運用**
- ❌ **連敗中のロット倍プッシュ** (マーチンゲール)
- ❌ **TPを伸ばす** (1:3 が頭打ち。1:5以上はPFが落ちる)
