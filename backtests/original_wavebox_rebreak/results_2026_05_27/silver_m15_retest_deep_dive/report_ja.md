## Key Findings

- Best total R was `0.15R / 2h`: 23 trades, 82.61% win rate, +12.99R, PF 3.49. This looks more practical than 0.25R because it keeps more trades while still improving execution quality.
- Best precision remains `0.25R / 2h`: 17 trades, 88.24% win rate, +11.42R, PF 4.76. This is a quality mode, not the only valid entry.
- Extending 0.25R wait from 2h to 3h added one winner and improved to 18 trades, 88.89%, +12.04R. Waiting beyond 3h did not add value in this sample.
- The two 0.25R/2h losses were both shallow H1 breaks. `trigger_atr >= 0.08` removed both losses, but reduced trades to 12, so it is a warning label rather than a confirmed production filter.
- Excluding entry hours 6 and 7 also removed both 0.25R losses and gave 13/13 winners, but this is very likely overfit until more OOS data exists.
- M15 bearish confirmation after retest did not clearly beat the simple retest. The best confirmation test was strong (+13.00R), but drawdown was larger and it adds complexity.

## Read

- A retest improves average risk and MAE, but it also skips trades. This is an execution-quality filter, not a new direction signal.
- If a filter removes both losses but also cuts too many winners, treat it as a warning label rather than a production rule.
- Confirmation after retest is tested separately because it may reduce false retests, but it can also enter later and give back the better price.
