## 初期設定

- 判定時間足: H4
- 本番判定: 確定足のみ
- V字速度: Balanced REC1.2
  - 下落が10本なら、61.8%〜80%回復まで12本以内を目安
- フィルタ: Strict 0.75-1.00 + width<=7
  - MACDヒストグラムが3本前より上昇
  - BB位置が0.75〜1.00
  - BB幅がATR7以下
- 騙し回避フィルタ: ON
  - BB位置が0.95以下
  - V候補からシグナルまでH4で16本以内
  - 単独rebreakでBB位置が0.95超、またはMACD slope3が0.03以下なら見送り
- エントリー: V候補後の高値停滞ブレイク、または押し後の戻り高値再ブレイク
- SL: V字の安値 - ATR x 0.25
- TP: シグナル終値基準でRR2.0
- 最大保有: H4で180本

## 回復比率スイープ

`回復本数 / 下落本数` を0.80〜2.00まで0.05刻み、さらに1.10〜1.30を0.01刻みで検証。

暫定の黄金比率は `1.20〜1.22`。実装値としてはシンプルに `1.20` を採用候補。

## TradingViewでの使い方

1. H4チャートで使う。
2. `判定時間足チャートでのみ売買する` はON推奨。
3. アラートは `Any alert() function call` を選ぶ。
4. 最初は必ずペーパートレードで確認する。
5. 実運用では、TradingViewの約定価格と実ブローカーの約定価格がズレるため、Signal/Entry/SL/TPの差を毎回記録する。

## 運用判断

この手法はTrendBreakV1とは別手法。TrendBreakV1は直近高安値更新のブレイク系、これは急落後V字候補を環境認識に使い、T5トリガーとMACD/BBで絞る反発継続系。
