# H4 V字回復 応用手法 仕様整理

作成日: 2026-05-30

## 目的

単なるV字検出ではなく、**急落後に大衆が売り切ったあと、買い戻しが本格化する局面**を数値化する。

初期V条件は、エントリー条件ではなく「売りの否定候補」として扱う。

## ベースV条件

- 時間足: H4
- 方向: ロング
- confirmed pivot high から confirmed pivot low までの急落
- 下落幅 >= 3.5ATR
- 終値が下落開始水準まで回復
- 回復本数 <= 下落本数
- シグナル足の実体比率 >= 60%
- 基本SL: 谷値 - 0.25ATR
- 基本TP: 2R

## 機能しやすい相場環境

### 1. 急落後のショートカバー

売りが一気に出たあと、戻りが下落より速い。
これは「売り方が踏まされている」状態で、Vの右肩速度が重要。

数値条件候補:

- 右肩速度 >= 左肩速度 x 1.2
- 回復本数 <= 下落本数
- 終値が左肩起点を0.05ATR以上上抜け

### 2. レンジ下限からの売り否定

レンジ下限を割ったあとにすぐ戻る形。
安値更新が継続しないため、買い戻しが入りやすい。

数値条件候補:

- V谷が60本レンジ下限から1.0ATR以内
- V後、谷を再更新しない
- 6本以内にレンジ中央値より上へ復帰

### 3. 強トレンド中の押し目回復

上位足の上昇トレンド中に、H4だけ急落して即回復する形。
押し目買いとして機能するが、すでに伸び切った場所は除外が必要。

数値条件候補:

- D1 close > D1 EMA50
- D1 EMA50 slope > 0
- H4 close が EMA50 から +4ATR以内

### 4. 初動前の売り失敗

強いトレンド途中ではなく、過熱していない場所で売りが否定され、その後に棚を作って上抜ける形。
今回の検証ではこの方向が最も実戦候補に近い。

数値条件候補:

- V前 ADX <= 26
- EMA50傾き <= 1.2ATR/20本
- 60本レンジ幅 <= 16ATR
- V後に6本棚を作り、棚高値を終値で上抜け

## 応用手法候補

### A. Clean V Reclaim

V字回復そのものを買う比較用ルール。

Entry:

- ベースV条件成立
- close >= 左肩起点 + 0.05ATR
- 実体比率 >= 60%
- 終値位置 >= 60%

除外:

- 回復本数 > 下落本数
- 右肩速度 < 左肩速度 x 1.2
- risk > 3.0ATR
- XAUUSDは別集計

用途:

- 単独採用ではなく、他ルールとの比較基準。

### B. V後の押し目買い

V字回復後、すぐ飛び乗らず、浅い押しを待つ。

Entry:

- ベースV条件成立後、12本以内
- 一度、左肩起点から0.25ATR以内まで押す
- ただしV谷から50%回復ラインを終値で割らない
- その後、直近3本高値を終値で0.05ATR上抜け

SL:

- 押し目安値 - 0.25ATR

TP:

- 1.5R / 2R / 2.5R / 3Rを比較

狙い:

- 飛び乗りの高値掴みを減らす。

### C. V後の棚ブレイク

今回の本命候補。Vを直接買わず、V後に上側で崩れないことを確認してから買う。

Entry:

- ベースV条件または緩和V条件成立
- V後、6本の棚を形成
- 棚幅 <= 1.8ATR
- 棚安値 >= V谷 + 下落幅 x 0.50 - 0.05ATR
- close > 棚高値 + 0.05ATR
- 実体比率 >= 40%
- 終値位置 >= 60%

SL:

- 棚安値 - 0.25ATR

TP:

- 1.5Rを主、2R以上は出口研究で比較

狙い:

- 「売りが否定された後、上側で滞在してから再点火」を取る。

### D. V後の高値更新ブレイク

V回復後に短期または中期高値を抜くまで待つ。

Entry:

- ベースV条件成立
- close > H4 Donchian20 high[1] + 0.05ATR
- または close > H4 Donchian55 high[1] + 0.05ATR
- 実体比率 >= 45%
- 終値位置 >= 60%

SL:

- V谷 - 0.25ATR または直近棚安値 - 0.25ATR

狙い:

- 大きく動くきっかけだけを拾う。

注意:

- Donchian55は大きく伸びやすいが件数が少ない。

### E. V + トレンドフィルター

押し目回復だけを狙うルール。

Entry:

- ベースV条件成立
- D1 close > D1 EMA50
- D1 EMA50 > D1 EMA200
- H4 EMA20 > H4 EMA50
- close > H4 EMA20

除外:

- H4 close - EMA50 > 4ATR
- D1 RSI > 70
- BB幅 > 8ATR

狙い:

- 既存上昇トレンド中の急落回復を取る。

### F. V失敗ショート

買いV字が失敗したところを売る逆利用。

Entry:

- ベースV候補が出る
- しかし左肩起点を終値で回復できない
- V後12本以内に、V谷 + 下落幅 x 0.50 を終値で下抜け
- MACD histogram < 3本前
- 実体比率 >= 45%

SL:

- 失敗戻り高値 + 0.25ATR

TP:

- 1.5R / 2R

狙い:

- 「買い戻しが入ったが続かなかった」場面の再下落。

## 除外条件

全候補共通で検証する。

- 回復本数 > 下落本数 x 1.2
- 下落幅 < 3.5ATR
- 右肩速度 < 左肩速度
- シグナル足の実体比率 < 40%
- 終値位置 < 60%
- risk > 3.0ATR
- V谷更新後、さらに0.10ATR以上安値更新した
- BB幅 > 8ATR
- 年末年始・主要休場
- 直近高値までの距離 < 1.0ATR
- 急落前からD1下降トレンドが強い:
  - D1 close < D1 EMA50
  - D1 EMA50 slope < 0
  - D1 ADX > 25

## 検証項目

- 勝率
- PF
- 総R
- 平均R
- 最大DD
- 最大連敗
- トレード回数
- 通貨ペア別成績
- 年別成績
- Research 2015-2024 / OOS 2025-2026
- H1 / H4 / D1比較
- Entry: 条件成立足終値 vs 次足始値
- TP: 1.5R / 2R / 2.5R / 3R
- SL: 谷 - 0.1ATR / 0.25ATR / 0.5ATR
- 24/48/72本後のMFE
- 48本以内に3ATR伸びた割合
- 72本以内に5ATR伸びた割合

## 最有望手法: H4 V Initial Shelf Breakout

現時点で最もPine化に向くのは、`C. V後の棚ブレイク` に `初動フィルター` を加えたもの。

理由:

- Vを直接買わないため、高値掴みを減らせる。
- 棚を作ることで「売りが否定された後に上側で滞在」を確認できる。
- PRECALMで、既存トレンドの途中乗りを減らせる。
- Python検証では、XAUUSD/CHFJPY/SILVER除外で 34 trades / 勝率58.82% / +15.55R / PF2.09 / DD5.11R。

### 完全ロジック仕様

名称:

- `H4 V Initial Shelf Breakout`

対象:

- H4チャート
- ロングのみ
- 初期対象: USDJPY, EURJPY, GBPJPY, AUDJPY
- 除外候補: XAUUSD, CHFJPY, SILVER

V条件:

- confirmed pivot high -> confirmed pivot low
- pivot幅: 3
- 下落幅 >= 2.8ATR
- 下落速度 >= 0.25ATR/本
- 右肩速度 >= 左肩速度 x 1.0
- 回復率 >= 65%
- 回復率 <= 125%
- 回復本数 <= 36本
- V谷後、V谷 - 0.10ATRを下抜けない

初動フィルター:

- V左肩起点時点のADX14 <= 26
- V左肩起点時点のEMA50傾き <= 1.2ATR/20本
- V左肩起点時点のClose - EMA50 <= 3.0ATR
- V左肩起点時点の60本レンジ幅 <= 16ATR

棚条件:

- V条件成立後、最大36本以内
- 直近6本を棚として評価
- 棚幅 <= 1.8ATR
- 棚安値 >= V谷 + 下落幅 x 0.50 - 0.05ATR

エントリー:

- close > 棚高値 + 0.05ATR
- 実体比率 >= 40%
- 終値位置 >= 60%
- 次足始値でロング

損切り:

- SL = 棚安値 - 0.25ATR
- risk <= 2.2ATR

利確:

- 初期版: TP = Entry + 1.5R
- 比較版: 2R / 2.5R / 3R
- 発展版: 1.5Rで半分利確、残りをEMA20割れまたはATR trail

同時保有:

- 同一通貨で1ポジションのみ
- ポジション保有中は新規シグナル無視

Pine化注意:

- pivotはconfirmedのみ使う
- `lookahead_off`
- signalとentryを分離する
- Python比較では、Pine成績より先に `entry_time` とシグナル本数一致を確認する
