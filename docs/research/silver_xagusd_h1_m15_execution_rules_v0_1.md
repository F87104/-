# SILVER/XAGUSD H1 Short + M15 Retest Execution v0.1

## 目的

SILVER/XAGUSDはH1で売りの発動を判断し、M15はより良い売値を待つ補助に限定する。
方向判断をM15へ落とさない。H1で条件が出たあと、M15で戻りを待てなければ見送りにする。

## 基本構造

狙いは「下落後の戻りが失敗し、H1の小さなBoxを下へ再ブレイクする場面」。
トレンド追随の初動ではなく、戻り売りの再加速を取る。

## H1発動条件

- 銘柄はSILVER/XAGUSD。
- 判断足はH1。
- H4は下方向。H4終値がH4 EMA100以下、かつH4 EMA100の傾きが下向き。
- 直近24本で0.8ATR以上の下落がある。
- 直近8本のBox幅が1.5ATR以内。
- 直近の戻しが0.6ATR以上ある。
- H1終値がBox安値を0.05ATR以上下抜ける。
- ブレイク足の値幅が0.5ATR以上。
- ブレイク足の実体率が85%以下。
- 終値が足の下側25%以内にある。
- entry hour 11/12は除外。
- 12月15日以降と1月10日以前は除外。

## 執行ルール

H1シグナル確定後、次のH1始値を基準価格にする。

- SLはBox高値 + 0.15ATR。
- 基準リスクRは `SL - 次H1始値`。
- TPはエントリー価格から1.2R。
- 最大保有は24本。
- 再エントリーなし。

### 本命: 0.15R戻り待ち

- 売り指値は `次H1始値 + 0.15R`。
- 待機は2時間。
- 2時間以内に約定しなければ見送り。
- 約定後はTP/SL/時間決済まで触らない。

### 厳選: 0.25R戻り待ち

- 売り指値は `次H1始値 + 0.25R`。
- 待機は3時間。
- 3時間以内に約定しなければ見送り。
- 件数は減るが、売値改善を優先するモード。

### 比較用: H1 open

- 次H1始値でそのまま売る。
- 検証比較用であり、実戦本命ではない。

## 危険サイン

危険サインはデフォルトでは除外せず、警告扱いにする。
サンプルが少ないため、最初から完全除外にすると過剰最適化になりやすい。

- 浅いブレイク: Box下抜け幅が0.08ATR未満。
- 危険時間: entry hour 6/7。

Pineでは `Warn only` と `Block` を切り替え可能。
実戦初期は `Warn only` で記録し、負けが集中することを確認してから `Block` に上げる。

## M15の使い方

M15はエントリー改善だけに使う。

- H1シグナルが出るまではM15を見ない。
- H1シグナル後、0.15Rまたは0.25Rの戻りをM15で待つ。
- M15で逆方向の新しい手法条件を追加しない。
- 戻りが来なければ追いかけて売らない。

## 検証での暫定評価

- H1 open: 27件、勝率77.78%、PF2.84。
- 0.15R戻り待ち: 23件、勝率82.61%、PF3.49。
- 0.25R戻り待ち: 17件、勝率88.24%、PF4.76。

0.15Rは実戦本命。0.25Rは厳選確認用。
ただし件数は少ないため、OOSとTradingView上の目視確認を必ず優先する。

## 実戦判断

- 通常は0.15R戻り待ちを使う。
- 重要局面、荒い相場、浅いブレイク、6/7時台は0.25Rまたは見送りを検討する。
- 約定しなかった場合は「良い売値が来なかった」と判断して追わない。
- 勝率よりも、負け方が浅いブレイクや危険時間に偏るかを記録する。
