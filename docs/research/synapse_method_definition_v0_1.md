# Synapse手法 定義整理 v0.1

作成日: 2026-05-25

このメモは、Synapse手法をPineで再現可能にするための一次仕様。
最初の目的は勝率・PFではなく、「人間が見て納得できる波、A、B、斜めライン、候補が出ること」。

## 1. 手法の核

Synapse手法は、単純なブレイク手法ではない。

狙うのは「2波の終わりから3波の始まり」。

見る順番は次の通り。

1. 親波・中波で、いまが2波の終端候補かを見る
2. その中に小波の三尊・逆三尊、またはそれに近い反転構造を見る
3. 反転構造に対して、水平線A、水平線B、斜めラインを引く
4. 止まる根拠と伸びる根拠が重なった候補だけを可視化する
5. エントリー条件は最後に追加する

重要なのは、波形認識とトレード判断を分けること。
まずは「ここがSynapse候補に見えるか」を可視化し、その後にエントリー可否を判定する。

## 2. 波の階層

Synapseでは、同じ反転構造を複数の波階層で見る。

### 親波

親波は、上位の方向と伸びしろを見るための波。

親波は直接エントリーを決めるものではない。
ただし、親波の方向と完全に逆らう候補は、初期段階では低品質として扱う。

ロングの場合:

- 親波の1波が上昇
- その後の押しが2波
- 2波が1波始点を明確に割らずに止まる
- 次に3波上昇へ移る余地がある

ショートの場合は上下を反転する。

### 中波

中波は、実際のセットアップを置く器。
ユーザーが「緑線」として見ている波は、この中波とする。

中波は、Pine上でも最優先で合わせる対象。
自動pivotが緑線の見え方と合わない場合は、エントリー条件を調整する前に中波定義を直す。

中波は3点で見る。

- `P0`: 中波1波の始点
- `P1`: 中波1波の終点
- `P2`: 中波2波の終点候補

ロングの場合:

- `P0` = 中波上昇1波の起点安値
- `P1` = 中波上昇1波の終点高値
- `P2` = 2波押しの終点候補安値
- ここから中波3波上昇を狙う

ショートの場合:

- `P0` = 中波下降1波の起点高値
- `P1` = 中波下降1波の終点安値
- `P2` = 2波戻りの終点候補高値
- ここから中波3波下降を狙う

### 小波

小波は、中波2波の終端付近に出る反転構造。
ここで水平線A、水平線B、斜めラインを決める。

小波は、必ず中波2波の中、または中波2波終端の近くにある必要がある。
中波がすでに3波方向へ伸び切った後に出る小波は、Synapse候補ではなく追いかけ候補として扱う。

## 3. A/B/斜めラインの定義

A/Bは高値・安値の最大最小で選ばない。
必ず役割で選ぶ。

### ロング候補

ロング候補は、中波2波の下落が終わり、3波上昇へ移る場面。
小波では逆三尊、または逆三尊に近い反転構造を見る。

構造の順番:

1. 中波2波の下落中に、戻り高値が作られる
2. その戻り高値から最後の売り波が出て、最終安値またはヘッドを作る
3. 最終安値から反発して、最初の戻り高値を作る
4. その後、右肩または押し目を作る
5. Bを上抜くと、3波候補として強くなる

ロングのA/B:

- `A`: 最終安値、つまりヘッドを作った最後の売り波の起点高値
- `B`: ヘッド後の最初の反発高値
- `斜めライン`: 中波2波の下落を支配していた戻り高値同士を結ぶ下降ライン。原則として、Aとその前の同種高値を結ぶ

Aは「最後の売りが始まった場所」。
Bは「最後の売りが止められた後、買いが最初に作った確認高値」。

B上抜けは反転確認。
A上抜けはより強い確認。
ただし、最初の可視化段階ではA上抜けを必須にしない。

### ショート候補

ショート候補は、中波2波の上昇が終わり、3波下降へ移る場面。
小波では三尊、または三尊に近い反転構造を見る。

ショートのA/B:

- `A`: 最終高値、つまりヘッドを作った最後の買い波の起点安値
- `B`: ヘッド後の最初の反落安値
- `斜めライン`: 中波2波の上昇を支配していた押し安値同士を結ぶ上昇ライン。原則として、Aとその前の同種安値を結ぶ

Aは「最後の買いが始まった場所」。
Bは「最後の買いが止められた後、売りが最初に作った確認安値」。

B下抜けは反転確認。
A下抜けはより強い確認。
ただし、最初の可視化段階ではA下抜けを必須にしない。

## 4. 候補の段階分け

Pineでは、最初からエントリーを出さない。
候補を段階で表示する。

### Stage 1: 中波2波候補

表示するもの:

- 親波
- 中波の緑線
- `P0`, `P1`, `P2`
- 中波2波の戻し率

条件:

- 中波が明確な `P0 -> P1 -> P2` を持つ
- 2波の戻しが浅すぎない
- 2波が1波始点を大きく壊していない

この段階では小波やA/Bは不要。

### Stage 2: 小波反転候補

表示するもの:

- 小波の三尊・逆三尊候補
- A
- B
- 斜めライン
- ヘッド/右肩

条件:

- 小波が中波2波の中、またはP2近辺にある
- ヘッドが中波2波の終端を説明している
- 右肩がヘッドを明確に更新し続けていない
- A/Bが役割で決まっている
- 斜めラインが中波2波の圧力線として自然に見える

### Stage 3: 確認済み候補

表示するもの:

- Bブレイク済み
- 斜めラインブレイク済み
- Bからの距離
- 中波2波からの回復率
- 見送り理由

条件:

- Bを抜いている
- 斜めラインも抜いている、またはほぼ同時に抜く形
- Bから離れすぎていない
- 中波3波方向にすでに伸び切っていない

この段階でも、まだ自動エントリーは出さない。
人間が見て納得できる位置に候補が出ることを確認してから、売買条件を追加する。

## 5. 見送り定義

Synapse候補として見送る場面:

- 中波がレンジで、2波から3波への転換に見えない
- 小波が中波2波の外にある
- ヘッドが中波2波の終端を説明していない
- A/Bが直近の最大最小で選ばれているだけで、役割線になっていない
- 斜めラインが中波2波の圧力線として機能していない
- Bを抜いた後にかなり走ってから候補が出る
- 中波3波方向に回復しすぎてから候補が出る
- 親波・中波・小波の向きがバラバラで、止まる根拠と伸びる根拠が重ならない

特に重要なのは、次の2つ。

- 伸び切った場所で出る印はSynapseではない
- レンジ中央で出る印はSynapseではない

## 6. Pine作り直しの方針

次のPineは、strategyではなくindicatorとして作る。

最初の版では、エントリー、SL、TP、勝率検証を入れない。

表示優先順位:

1. 中波の緑線が人間の見方と合うこと
2. 中波の `P0/P1/P2` が分かること
3. 小波反転が中波2波の終端付近だけに出ること
4. A/B/斜めラインが役割通りに出ること
5. Bブレイク、斜めブレイク、遅すぎ判定を別表示にすること

入力は最小限にする。

- 大波pivot幅
- 中波pivot幅
- 小波pivot幅
- 中波2波の最小/最大戻し
- P2近辺許容ATR
- Bブレイク後の追いかけ上限ATR
- 中波回復率上限

スコア、ロット、strategy注文、TP/SLは後回し。

## 7. 現行版から引き継ぐもの・捨てるもの

引き継ぐもの:

- 親波/中波/小波の3階層
- 緑線を中波とする考え
- A/Bを最大最小ではなく役割で決める考え
- 候補を一定期間保持する考え
- Bから離れすぎた候補を見送る考え

一度捨てるもの:

- strategy注文
- ロット判定
- 複雑なスコア
- 緩和モードの多段階化
- PFや勝率を前提にした条件調整
- ラベルだらけになる詳細表示

## 8. 次の実装チェックリスト

最初のPineで確認すること:

- 緑の中波が、ユーザーの見ている中波に近いか
- Aが「最後の波の起点」になっているか
- Bが「ヘッド後の確認線」になっているか
- 斜めラインがA/Bの代用品ではなく、2波の圧力線になっているか
- 候補が中波2波の終端付近に出るか
- 伸び切った後に候補が出ないか
- レンジ中央に候補が出ないか

このチェックが通ってから、エントリー条件を段階的に追加する。
