---

## 0. なぜこの戦略が機能するか（理屈）

V 字回復には、儲かるパターンとそうでないパターンの 2 種類があります。

### パターン①: 本物の V 字 → 上がる ✅

```
   ↑
    \         /
     \       /
      \     /
       \___/    機関が買い戻している
```

- **強いトレンドの押し目**で発生
- 急落 → **急回復**（左右の波が対称）
- 大衆が「もうダメだ」と売った直後に機関が拾いに行く
- → V 字後にトレンドが再開する

### パターン②: ニセの V 字 → 下がる ❌

```
    \    
     \  /\
      \/  \
           \      一瞬戻っただけで再下落
            \
```

- **下降トレンドの一時的な反発**で発生
- 急落 → **だらだら戻る**（右肩が遅い）
- いわゆる **「デッドキャットバウンス」** (死に体反発)
- → 続落して負ける

---

### 形だけ見ても①と②は区別できない

ここが大事なところ。チャートだけ見て「V 字だ！」と買うと、半分くらいは ② を掴まされて負けます。

そこで **3 つの "外部条件" でフィルタ** をかけます。

---

## 1. 3 つのフィルター

### フィルター① — 上位足 (D1) が上向き 【最重要】

> 「上位足が逆行している時に押し目買いはしない」これだけで勝率 +5〜15%

| D1 の状態 | 判定 |
|---|---|
| 価格が D1 EMA50 より上 | ✅ |
| D1 EMA50 が EMA200 より上 | ✅ |
| D1 EMA50 が直近 10 日で上向き | ✅ |
| D1 RSI ≥ 35（売られすぎでない） | ✅ |

→ **すべて ✅ の時しかエントリーしない**

---

### フィルター② — V 字の "綺麗さ" チェック

| 項目 | 条件 | 意味 |
|---|---|---|
| **急落の大きさ** | 3.5 ATR 以上 | 中途半端な下げは無視 |
| **右肩の速さ** | 左肩より **1.2 倍速い** | だらだら戻りは無視（= ②除外） |
| **右肩の長さ** | 左肩以下の本数で戻る | 機関の急いだ買いを確認 |
| **シグナル足** | 実体 60% 以上の強い陽線 | 当日に買い圧力が集中 |
| **終値位置** | 高値から 60% 以上 | 引け値が高い = 引け強さ |

**注:** 「右肩 ≥ 左肩 × 1.20」が最大のキモ。**ダラダラ戻り (= ニセ V)** をここで弾きます。

---

### フィルター③ — 場所もチェック

| 項目 | 条件 | 意味 |
|---|---|---|
| **V 底が D1 EMA50 の近く** | ±1 ATR 以内 | 正しい押し目位置 |
| **直近高値まで距離あり** | 1.5R 以上 | TP 到達余地あり |
| **ボラがある** | ATR が長期平均の 60% 以上 | 動かない相場で発注しない |
| **年末年始でない** | 12/15〜1/10 はスキップ | 流動性歪み回避 |

## 2. エントリー・損切り・利確のルール

```
                      ★ TP = +2R       ← ここで利確
                      │
                      │
              ────────●━━━━━━ Entry  ← 次の足の始値で買う
              │
              │
              │       ↓ Risk (1R)
              │
              ●━━━━━━ SL = V 底 − 0.25 ATR    ← ここで損切り
            V 底
```

| | 値 |
|---|---|
| **Entry** | シグナル足の **次のローソク始値** |
| **SL（損切り）** | V 底 − 0.25 ATR |
| **TP（利確）** | Entry + **2R**（リスクの 2 倍） |
| **時間切れ** | 180 本（H4 で約 1 ヶ月）保有したら手仕舞い |

**R = 1 トレード当たりリスク** のこと。SL から Entry までの距離 = 1R。

## 3. 期待される成績（目安）

> あくまで目安。実際は通貨・期間で変動するので、必ず自分で BT で確認。

| 指標 | 目安 | 最低ライン |
|---|---|---|
| 勝率 | **55〜62%** | 50% 以上 |
| プロフィットファクター (PF) | **1.6〜2.2** | 1.3 以上 |
| 平均 R | **+0.4〜+0.7** | +0.2 以上 |
| トレード数 | **年 8〜15 回 / 通貨** | 年 5 回以上（少なすぎる戦略は信用しない） |
| 最大連敗 | **4〜6 回** | 9 回以下 |
| 最大 DD | **6〜10R** | 15R 以下 |

**6 通貨で同時運用**すれば、年 50〜90 トレード、DD が平準化されてもっと安定。

## 5. やってはいけないこと

### ❌ NG 行動

1. **D1 が下向きなのに「V 字だ！」で買う**
   → ②のニセ V を踏む

2. **右肩がだらだら（左肩より遅い）でも買う**
   → デッドキャット確定

3. **直近高値の真下で買う**
   → TP 届かず横ばい

4. **イベント前（NFP/CPI/FOMC）に新規エントリ**
   → ギャップで SL すら効かない

5. **年末年始に動かす**
   → 流動性歪みで誤シグナル多発

6. **シグナル足の終値で焦って買う**
   → このストラテジは **次足始値** が前提

## 7. もし結果が悪かったら（トラブルシューティング）

### 症状 1: トレード数が極端に少ない（年 3 回以下）

| 対処 | パラメータ |
|---|---|
| 急落の最小幅を緩める | `minDropAtr` 3.5 → 3.0 or 2.5 |
| 右肩速度を緩める | `minSpeedRatio` 1.20 → 1.10 |
| EMA 距離を緩める | `vNearEmaAtr` 1.0 → 1.5 |

### 症状 2: 勝率が低い（50% 未満）

| 対処 | パラメータ |
|---|---|
| シグナル足品質を厳しく | `minBodyRatio` 0.60 → 0.70 |
| 右肩速度を厳しく | `minSpeedRatio` 1.20 → 1.50 |
| D1 RSI 下限を上げる | `htfRsiAvoid` 35 → 40 |

### 症状 3: PF が低い (1.3 未満)

| 対処 | パラメータ |
|---|---|
| TP を伸ばす | `tpR` 2.0 → 2.5 |
| SL を縮める | `stopBufferAtr` 0.25 → 0.10 |
| 直近高値距離を厳しく | `nearHighMinR` 1.5 → 2.0 |

### 症状 4: 最大連敗が多い (10 回以上)

→ HTF フィルタが緩い可能性。`useTrendFilter` ON を確認。
→ 通貨を絞る（XAUUSD / GBPJPY を外す等）。
