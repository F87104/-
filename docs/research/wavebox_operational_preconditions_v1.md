# USDJPY H1 WaveBox Operational Preconditions v1

作成日: 2026-05-26

## 目的

WaveBox USDJPY H1 Rebreak を実戦投入する前に、前提条件を固定する。

この文書の目的は、勝率やPFを上げることではない。

目的は以下。

- 検証結果を過信しない
- TradingViewとPythonのズレを潰す
- 時間フィルターの前提を曖昧にしない
- 実戦で「都合よく解釈する余地」を減らす

## 最重要結論

この手法は、現時点では `実戦候補` であって、まだ完成済みの実弾手法ではない。

過去検証では強いが、以下の弱点がある。

- 2023年以降のUSDJPY相場にかなり助けられている
- OOS 2025-2026は6件しかない
- 時間フィルター依存が大きい
- `GO A+ or clean A` は38件で、まだ標本が少ない

したがって、期待値はPF3ではなく、まずPF1.5-2.0程度に割り引いて見る。

## 固定する前提

### 銘柄

- 対象は `USDJPY` のみ。
- 他通貨へ横展開しない。
- GBPJPY、AUDJPY、CHFJPYなどは別手法として再検証する。

理由:

- クロスシンボル検証では、全通貨共通手法として扱う根拠が弱い。
- USDJPYだけに絞って、Pine表示・フォワード記録・実運用ルールを完成させる。

### 時間足

- メイン時間足は `H1` のみ。
- M5/M15/M30はエントリー足として使わない。
- H4は環境認識として使うが、絶対条件にはしない。

理由:

- 初回比較でH1だけが残った。
- H4逆行なしは心理的には扱いやすいが、A+より優先しない。

### 手法の型

これはV字底当てではない。

ロングなら以下の形だけを狙う。

1. 強い上昇1波
2. 50%-78.6%の押し
3. 押し目の終端付近で8本Box停滞
4. Box上限を終値で再ブレイク
5. 次のH1始値でENTRY

ショートは上下反転。

狙うのは `強い1波の後の押し目停滞からの3波再加速`。

## 検証標準条件

標準は以下で固定する。

- Rule preset: `v0.4 filtered`
- Wave1 quality: `Strict`
- Pivot幅: 3
- Pivot間最小スイング: 1.15ATR
- 1波値幅: 3.0ATR以上
- 1波速度: 0.35ATR/本以上
- 1波時間: 24本以内
- 戻し: 50%-78.6%
- Box: 直近8本
- Box幅: 2.0ATR以内
- ブレイク余白: 0.05ATR
- 追いかけ上限: 0.80ATR
- ブレイク足実体率: 0.25以上
- EMA20方向一致
- TP: 1.5R
- SL: P2またはBox外側 + 0.2ATR
- 最大保有: 72本
- 年末年始除外: 12/15以降、1/10以前

## 時間フィルター前提

時間フィルターは `SIGNAL時刻` ではなく、`ENTRY予定時刻` で判定する。

H1では、SIGNAL足の `time_close` が次のH1足の始値時刻に相当する。

例:

- 00:00-01:00の足でSIGNAL確定
- 次の足、01:00始値でENTRY予定
- entry hour = 1
- 1時除外に該当

### 標準除外

検証標準では以下を除外する。

- 1時
- 6時
- 14時

### 根拠の強さ

| 時間 | 扱い | 検証上の根拠 |
| --- | --- | --- |
| 1時 | 必須除外 | 21件 / 勝率14.29% / -13.59R / PF0.24 |
| 6時 | 必須除外 | 6件 / 勝率16.67% / -3.71R / PF0.28 |
| 14時 | 暫定除外 | 2件 / 勝率50.00% / +0.46R / PF1.45 |

判断:

- 1時と6時は必須除外。
- 14時は標本が少ないため、強い根拠とは言わない。
- ただし標準検証は `1/6/14除外` で作られているため、フォワード20件までは外さない。
- 14時を外すか残すかは、別枠で `1/6のみ除外` として比較する。

### タイムゾーン前提

時間フィルターはタイムゾーンがズレると意味が壊れる。

必須確認:

- Python検証CSVの時刻基準
- TradingView銘柄フィードの時刻基準
- Pineの `brokerTz`
- TradingViewチャート下部の表示タイムゾーン

実戦前に、過去シグナルを最低5件照合する。

照合項目:

- Pythonの `signal_time`
- TradingViewのSIGNAL足
- Pythonの `entry_time`
- PineのENTRYマーカー
- entry hourが一致するか

この照合が終わるまで、時間フィルター込みの成績を信用しない。

## データ前提

過去検証データ:

- USDJPY M5 CSV
- M5からH1へ集約
- 2014-01-01から2026-05-22付近まで

注意:

- TradingViewのMATSUI/FXCMデータとは完全一致しない可能性がある。
- 高値/安値が少し違うだけで、Box、ATR、SL、TP、ブレイク判定が変わる。
- したがって、PythonとTradingViewの一致確認は必須。

許容できる差:

- SIGNAL時刻: 一致が原則
- 方向: 完全一致
- Rank/Action: 完全一致が原則
- Entry: 1-2pips程度まで
- Stop/Target: 1-3pips程度まで

許容できない差:

- TradingViewだけに出るSIGNAL
- Pythonだけに出るSIGNAL
- 方向違い
- A+/A/B/Action違い
- Entry時刻違い

## 実戦前チェック

実弾前に、以下をすべて満たすこと。

### Pine設定

- USDJPY H1で使っている
- Pineは `wavebox_usdjpy_h1_rebreak_v1_2.pine` を使う
- Rule preset: `v0.4 filtered`
- Wave1 Quality: `Strict`
- `strategy注文` はOFF
- 内部R対象は `GO only`
- Bシグナルは観察のみ
- `brokerTz` を検証CSV基準に合わせた
- `TZ照合済み（Python/TV）` は、過去シグナル照合後にだけONにする
- `WaveBox SIGNAL GO` アラートを使う

### エントリー条件

実戦候補は以下のみ。

- `GO A+`
- `GO A`

以下は実戦初期では打たない。

- `SELECT A`
- `OBS B`
- `SKIP`

### 目視条件

打つ前に、チャート上で以下が見えること。

- 強い1波がある
- その後の押しが50%-78.6%に収まる
- 押しの終端付近でBox停滞している
- Boxを終値で再ブレイクしている
- 伸び切った後の追いかけではない

## 実行前の見送り条件

Pineが `GO A+` または `GO A` を出しても、以下なら見送る。

- 重要指標直前
- スプレッド異常
- 次足始値が大きく飛んだ
- SL幅が通常より極端に広い
- TradingViewとPythonの照合が未完了
- 時間フィルターのタイムゾーンが未確認
- 直近で3連敗してレビュー中
- 月間 -3R 到達後

## リスク前提

フォワード検証中:

- 1回リスク: 最大0.25%
- 30件到達までは最大0.5%を超えない
- A+だけロットを上げない
- 複数ポジションを同時に持たない
- 途中でパラメータを変えない

実戦昇格後:

- 30件以上のフォワード記録が必要
- Python/Pine差異が許容範囲内
- `GO A+ / GO A` の実績が大崩れしていない
- 目視で納得できるシグナルが多い

## 停止条件

以下で停止する。

- 3連敗
- 月間 -3R
- PythonとTradingViewのSIGNAL時刻がズレる
- Entry/SL/TPがズレる
- 連続して「見た目が悪いGO」が出る
- 重要指標・スプレッドで実行不能が続く

停止後は、最低5件のスクリーンショットを見直す。

## 検証結果の見方

過去検証の代表値:

- Strict all: 69件 / 勝率65.22% / PF2.69
- Strict A+ or clean A: 38件 / 勝率68.42% / PF3.15
- Strict 2014-2022: 54件 / 勝率59.26% / PF2.08
- Strict 2023+: 15件 / 勝率86.67% / PF9.42
- 時間フィルターなし: 98件 / 勝率51.02% / PF1.51

見方:

- PF3は期待しない。
- 実力はPF1.5-2.0程度に割り引く。
- 2023年以降の好調を手法の実力と断定しない。
- OOS 6件全勝は強い証拠ではない。

## 合格条件

最初の合格条件:

- `GO A+ / GO A` を20件記録
- すべてのSIGNAL/ENTRYがTradingViewとPythonで照合済み
- 時間フィルターのタイムゾーンが確認済み
- 目視で「押し目停滞からの再ブレイク」に見える
- PFが大崩れしていない

本格採用条件:

- `GO A+ / GO A` を30件以上記録
- 見送り理由が一貫している
- 連敗時にルールを変えずに運用できている
- 実運用コスト込みでも期待値が残る

## 現時点の判定

現時点では、実弾本格運用はまだ不可。

許可するのは以下。

- TradingViewでの目視確認
- フォワード記録
- 小ロットまたはデモでの `GO A+ / GO A` 限定確認

禁止するのは以下。

- PF3を前提にロットを上げる
- SELECT AやBを裁量で打つ
- 途中で時間フィルターを変える
- USDJPY以外へ同じ条件で流用する
