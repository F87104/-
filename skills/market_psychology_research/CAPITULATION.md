# Capitulation — 降伏

> 長期間含み損に耐えていた参加者が、最後に一斉に投げ、
> その投げ切りが反転の起点になる構造を研究する。

[← SKILL.md に戻る](SKILL.md)

---

## 1. 概要

Capitulation（降伏）とは、**長く含み損に耐えてきた参加者が「もう耐えられない」と一斉に投げ、
その投げ切りが反転の起点になる構造** である。

[Long Liquidation](LONG_LIQUIDATION.md) が「直近で買った参加者の投げ」だとすれば、
Capitulation は **「長期保有者の最後の投げ」** である。
含み損に耐えていた最後の集団までが降伏すると、売り手が枯渇し、価格は反転する。

降伏は底（または天井）の形成と密接に結びつく。
「最後の売りが出尽くした」地点が反転点になるという、典型的なクライマックス構造である。

---

## 2. 市場参加者の心理

- **期待していた人**: 「いつか戻る」と信じて含み損を長期間保有してきた参加者。
- **失望の瞬間**: 下落が止まらず、含み損が許容限界を超える。希望が断たれる。
- **損切りする人**: 最後まで耐えていた長期保有者。投げが一斉・大量に発生する。
- **加速の方向**: 投げ切りの直後に **反転**（投げが終われば売り手が枯渇する）。
- **油断していた人**: 「もう底だろう」と早めに拾い、追加の投げに巻き込まれた参加者。

> 核心: **最後の投げが出尽くした瞬間こそ、反転の起点。**

---

## 3. チャート上の構造

```
長期間含み損（だらだらとした下落・保有の継続）
  ↓
最後の投げ（パニック的な投げ売り＝クライマックス）
  ↓
反転（売り枯渇 → 急反発）
```

- クライマックスでは **長いヒゲ**（投げ切りと即座の買い戻し）が出やすい。
- ATR が急拡大し、出来高が急増する（感情のピーク）。
- 反転後は V字に近い戻りを伴うことが多い。

---

## 4. 数値化できる特徴

| 特徴量 | 観測の狙い |
|---|---|
| **ATR急拡大** | 投げのクライマックスで ATR が直近平均の何倍に跳ねたか |
| **長いヒゲ** | 下ヒゲ/実体比（投げ切りと即買い戻しの痕跡） |
| **出来高急増** | 出来高が直近平均の何倍か（降伏の規模） |
| **下落の累積期間** | 含み損が積み上がった期間の長さ（降伏のたまり具合） |
| **反転確認** | クライマックス後の終値が前バー高値を上抜けたか |

---

## 5. 検証仮説

- **H1**: 長期下落の末に、ATR急拡大 + 長い下ヒゲ + 出来高急増が同時に発生すると、
  その後 M バーで反転（上昇）が有意に観測される。
- **H2**: 3条件（ATR・ヒゲ・出来高）が揃ったクライマックスは、
  単一条件のみより OOS での反転再現性が高い。
- **H3**: 降伏前の下落期間が長いほど、反転後の戻り幅が大きい。

## 6. Python実装候補

- ATR比スパイク検出: `atr / atr.rolling(N).mean()` が閾値を超えたバー。
- ヒゲ比率: `(min(open, close) - low) / (high - low)` で下ヒゲの長さを正規化。
- 出来高スパイク: 出来高 / 直近平均。
- 下落累積: ピークからの下落バー数・下落率。
- イベントスタディ: クライマックス検出後の反転リターンを OOS で集計。

```python
# 方針（疑似コード）
def detect_capitulation(df, atr_mult, wick_thr, vol_mult, fall_lb):
    atr_spike = df.atr[-1] > atr_mult * df.atr.rolling(fall_lb).mean()[-1]
    long_wick = lower_wick_ratio(df.iloc[-1]) > wick_thr
    vol_spike = df.volume[-1] > vol_mult * df.volume.rolling(fall_lb).mean()[-1]
    prolonged_fall = bars_since_peak(df) >= fall_lb
    return atr_spike and long_wick and vol_spike and prolonged_fall
```

## 7. Pine Script実装候補

- ATR比・下ヒゲ比・出来高比を同時に判定し、クライマックスバーを `plotshape` でマーク。
- 長期下落期間を `ta.barssince` で計測。
- 降伏候補に `bgcolor` でハイライト。
- `alertcondition` で降伏クライマックスを通知（観測用）。

- 「通常の投げ（[Long Liquidation](LONG_LIQUIDATION.md)）」と「最後の投げ（降伏）」の定量的な境界。
