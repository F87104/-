# Dormant Breakout — 長期間放置された節目の更新

> 数ヶ月〜数年触られていない節目が更新され、
> そこに紐づく損益ラインが一斉に変化することで加速が起きる構造を研究する。

[← SKILL.md に戻る](SKILL.md)

---

## 1. 概要

Dormant Breakout（休眠ブレイク）とは、**数ヶ月〜数年触られていなかった節目が更新され、
そこに紐づく損益ライン（含み損益の境界）が一斉に変化することで加速が起きる構造** である。

長期間放置された高値・安値は、その間「忘れられた節目」である。
その水準には、過去にそこで売買した参加者の損益分岐や、長期の指値・逆指値が眠っている。
価格がこの休眠した節目を更新すると、

- 長期保有者の損益が反転する（含み益 ↔ 含み損）
- 長く機能していた指値・逆指値が一斉に約定する
- 「ここは抜けない」という前提が崩れる

ことで、通常のブレイクより大きく加速しやすい。
日々試されている節目（みんなが見ているライン）とは異なり、
休眠節目のブレイクは [Trap](TRAP.md)（だまし）になりにくいという仮説も研究対象とする。

---

## 2. 市場参加者の心理

- **期待していた人**: 「この長期高値/安値は抜けない」と信じていた長期の参加者。
- **失望の瞬間**: 何年も守られた節目が破られる。長年の前提が崩壊する。
- **損切りする人**: 節目を背にポジションを持っていた長期保有者。損益反転で投げを迫られる。
- **加速の方向**: ブレイクした方向。眠っていた指値・逆指値と長期保有者の投げが燃料になる。
- **油断していた人**: 「どうせ抜けない」と高をくくっていた参加者ほど、ブレイクの初動に乗り遅れる。

> 核心: **長く触られなかった節目ほど、更新時に大きな損益の地殻変動を起こす。**

---

## 3. チャート上の構造

```
数ヶ月〜数年触られていない高値（休眠した節目）
  ↓
更新（休眠節目を終値でブレイク）
  ↓
損益ライン変化（長期保有者の含み損益が反転）
  ↓
加速（眠っていた注文と投げが連鎖し大きく走る）
```

- 節目が「いつ最後に触られたか」が重要（休眠期間が長いほど効果が大きい仮説）。
- ブレイク後、節目が今度はサポート/レジスタンスとして機能転換することが多い。

---

## 4. 数値化できる特徴

| 特徴量 | 観測の狙い |
|---|---|
| **休眠期間** | その高値/安値が最後に触られてからの経過バー数（休眠の長さ） |
| **節目の鮮度** | 直近で何度も試されていない（タッチ回数が少ない）こと |
| **更新確定** | 終値で休眠節目を超えたか |
| **ブレイク後ボラ** | 更新後の ATR 拡大（地殻変動の規模） |
| **機能転換** | ブレイク後に節目がサポート/レジスタンスへ転換したか |

---

## 5. 検証仮説

- **H1**: 休眠期間が長い（例: ≥ X ヶ月）節目を終値ブレイクすると、
  休眠期間が短い節目のブレイクより、その後 M バーの加速が大きい。
- **H2**: 休眠節目のブレイクは、頻繁に試される節目のブレイクより [Trap](TRAP.md)（だまし）率が低い。
- **H3**: 休眠期間とブレイク後のトレンド持続には正の相関がある。

## 6. Python実装候補

- 休眠節目の抽出: スイングハイ/ローを検出し、各水準が最後に触られてからの経過期間を計測。
- 休眠スコア: 休眠期間 × （直近のタッチ回数の少なさ）。
- ブレイク検出: 終値が休眠節目を超えたイベントを記録。
- イベントスタディ: ブレイク後のリターン・トレンド持続を **休眠期間で層別** して OOS 集計。
  だまし率も併せて測定する。

```python
# 方針（疑似コード）
def detect_dormant_breakout(df, min_dormant_bars, swing_lb):
    levels = find_swing_levels(df, swing_lb)         # 過去のスイング高安
    for lv in levels:
        dormant = bars_since_last_touch(df, lv) >= min_dormant_bars
        broke = df.close[-1] > lv.price if lv.is_high else df.close[-1] < lv.price
        if dormant and broke:
            yield lv
```
