# Expectation Failure — 期待崩壊

> すべての心理構造の起点。
> **市場参加者が期待している方向へ、価格が進まなくなった瞬間** を研究する。

[← SKILL.md に戻る](SKILL.md)

---

## 1. 概要

Expectation Failure（期待崩壊）とは、**ある方向への値動きが続くと信じている集団の期待が、
価格によって裏切られる瞬間** を指す。

下落が続いている局面では、多くの参加者が「続落するはず」と期待する。
その期待に基づいて新規の売りが入り、戻りは売られる。
ところが、ある地点で価格が **続落しなくなる**。
この「期待が満たされない瞬間」こそが、後続のあらゆる加速（Short Squeeze, FOMO 等）の母体となる。

本研究所では、この期待崩壊を **最も基礎的な観測対象** と位置づける。
他のすべてのパターンは、突き詰めれば「誰の、どの方向への期待が崩壊したか」で説明できる。

---

## 2. 市場参加者の心理

- **期待していた人**: トレンド追随の参加者。下落局面なら「続落」を、上昇局面なら「続伸」を期待。
- **期待の内容**: 「直近の勢いはまだ続く」という連続性への信頼。
- **失望の瞬間**: 期待した方向に新値が出ない（高値/安値を更新できない）。戻り売り・押し目買いが機能しない。
- **損切りする人**: 期待方向に新規参入した直近の参加者。含み損が膨らみ、逆方向への損切りを迫られる。
- **加速の方向**: 崩壊した期待と **逆方向**。続落を期待していた集団が踏み上げられれば上昇が加速する。

> 核心: **「価格が、期待された方向へ進むのを止めた瞬間」** を捉える。

---

## 3. チャート上の構造

```
急落
  ↓
続落期待（新規売り増加・戻り売り）
  ↓
続落しない（安値を更新できない）
  ↓
上昇（期待の裏切り＝期待崩壊の確定）
```

- 直近の勢いと同方向に新値が出続ける限り、期待は満たされている（崩壊なし）。
- 新値更新が止まり、逆方向の足が出始めた地点が **期待崩壊の候補**。
- 崩壊が確定すると、しばしば V 字・戻り売り失敗・出来高増加を伴う。

---

## 4. 数値化できる特徴

| 特徴量 | 観測の狙い |
|---|---|
| **V字回復** | 急落幅に対する戻り幅の比率（例: 戻り / 下落 ≥ 0.6）で期待崩壊の強さを測る |
| **戻り売り失敗** | 直近高値を試した後に下落できず、逆に上抜けた回数 |
| **高値更新** | 期待された下落方向ではなく、上方向に新値が出たか |
| **出来高増加** | 崩壊地点での出来高が直近平均の何倍か |
| **ボラ拡大** | ATR が崩壊前後で拡大したか（期待の急転換のサイン） |

---

## 5. 検証仮説

- **H1**: 急落後に安値更新が N バー止まり、かつ V字戻り比率が閾値を超えると、
  その後の M バーで上方向への加速が有意に観測される。
- **H2**: 戻り売りが2回以上失敗（高値を更新）した地点は、単純な安値反発より OOS での再現性が高い。
- **H3**: 期待崩壊時の出来高スパイク（直近平均比 ≥ X 倍）は、加速の持続性と正の相関を持つ。

いずれも反証可能な形で、OOS を含む期間で検証する。

## 6. Python実装候補

- 安値更新の停止検出: 直近 N バーで安値が更新されていないことを判定する関数。
- V字回復スコア: 急落区間と戻り区間を自動分割し、戻り比率を算出。
- 戻り売り失敗カウンタ: スイングハイの更新試行回数とその成否を集計。
- イベントスタディ: 期待崩壊シグナル後の +1〜+M バーのリターン分布を OOS で集計し、
  ベースライン（無条件）と比較する。

```python
# 方針（疑似コード）
def detect_expectation_failure(df, lookback, v_ratio_thr, vol_mult):
    drop = recent_drop(df, lookback)            # 直近の下落幅
    no_new_low = not made_new_low(df, lookback) # 安値更新が止まった
    v_score = recovery / drop                   # V字戻り比率
    vol_spike = df.volume[-1] > vol_mult * df.volume.rolling(lookback).mean()[-1]
    return no_new_low and v_score >= v_ratio_thr and vol_spike
```

## 8. 今後の研究課題

> **検証済みの知見** ([検証レポート](verification/VERIFICATION_EXPECTATION_FAILURE_SQUEEZE.md)):
> まったく同じ「急落 → V字回復」という期待崩壊の地点でも、
> **完全V字回復を待って飛び乗るだけ（構造確認なし）は PF 0.72 の負け**。
> **棚（踏み上げ構造）を確認してから入ると PF 2.55 の勝ち**。
> → 期待崩壊は「地点」ではなく、その後の **踏み上げ構造とセット** で初めてエッジになる。

- **事例研究**: 現在進行中の **H4 V Initial Shelf Breakout** を期待崩壊の代表事例として扱う。
  急落 → 続落期待 → 続落停止 → V字 → 初期棚（shelf）形成 → 棚ブレイクという流れは、
  「続落期待の崩壊」と「踏み上げ（Short Squeeze）の起点」が連続する典型例である。
- 期待崩壊の「確定」をどのバーで宣言すべきか（早すぎる確定はノイズ、遅すぎると優位性が消える）。
- 上昇トレンドにおける対称形（続伸期待の崩壊 → Long Liquidation）との統一的な定式化。
- 崩壊の強度を連続値でスコア化し、Short Squeeze / Trap / FOMO への分岐確率を推定する。
