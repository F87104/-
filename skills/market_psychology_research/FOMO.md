# FOMO — 乗り遅れ恐怖

> 押し目のない上昇に焦った参加者が追随買いを入れ、
> その追随が更なる上昇を生む自己強化的な構造を研究する。

[← SKILL.md に戻る](SKILL.md)

---

## 1. 概要

FOMO（Fear Of Missing Out / 乗り遅れ恐怖）とは、**押し目のない上昇に焦った参加者が
「乗り遅れたくない」という恐怖から追随買いを入れ、その追随が更なる上昇を生む自己強化的構造** である。

通常の上昇には押し目（一時的な下落）があり、参加者はそこで乗ることを期待する。
ところが押し目が来ないまま価格が上がり続けると、待っていた参加者は焦り始める。
「もう待てない」と感じた集団が高値を承知で飛び乗り、その買いが価格を更に押し上げる。
これが繰り返されることで、合理性を欠いた急騰（パラボリック）が形成される。

FOMO は自己強化的だが、押し目を作らないがゆえに **脆い**。
燃料（新規の買い手）が尽きると、しばしば [Long Liquidation](LONG_LIQUIDATION.md) へ反転する。

---

## 2. 市場参加者の心理

- **期待していた人**: 押し目を待っていた参加者。「いつか下がったら買おう」と構えていた。
- **失望の瞬間**: 押し目が来ない。待っているうちに価格がどんどん離れていく。
- **行動の転換**: 失望が「乗り遅れの恐怖」に変わり、高値でも飛び乗る。
- **加速の方向**: **上方向**。追随買いが買いを呼ぶ自己強化ループ。
- **油断していた人**: 「さすがに高い、すぐ下がる」と逆張りした参加者は踏み上げられる。

> 核心: **「下がらないこと」への焦りが、更なる上昇の燃料になる。**

---

## 3. チャート上の構造

```
上昇
  ↓
押し目がない（浅い調整すら来ない）
  ↓
焦る（待機していた参加者の心理転換）
  ↓
追随買い（高値での飛び乗り）
  ↓
加速（買いが買いを呼ぶ自己強化ループ＝パラボリック）
```

- 角度が徐々に立ち上がる（パラボリックな上昇）。
- EMA からの上方乖離が拡大し続ける。
- 高値更新が連発し、調整らしい調整が入らない。

## 4. 数値化できる特徴

| 特徴量 | 観測の狙い |
|---|---|
| **高値更新連発** | 連続して直近高値を更新したバー数・頻度 |
| **EMA乖離** | 終値と EMA の乖離率（追随買いの過熱度） |
| **ボラ拡大** | ATR が上昇とともに拡大しているか（加速のサイン） |
| **押し目の浅さ** | 直近の最大押し幅（小さいほど FOMO 的） |
| **上昇角度** | 単位バーあたりの上昇率の加速（パラボリック度） |

---

## 5. 検証仮説

- **H1**: EMA乖離率が閾値を超え、かつ N バー連続で高値更新が続くと、
  その後も短期的に上昇が継続する（FOMOの自己強化）一方、過熱ピーク後の反転リスクが高まる。
- **H2**: 押し目の浅さ（最大押し幅の小ささ）は、その後の反転（[Long Liquidation](LONG_LIQUIDATION.md)）の鋭さと正の相関を持つ。
- **H3**: EMA乖離 + ボラ拡大が同時成立した過熱は、単一条件より反転の OOS 再現性が高い。

## 6. Python実装候補

- EMA乖離率: `(close - ema) / ema` を算出。
- 高値更新連発: 直近 N バーでの高値更新回数。
- 押し目の浅さ: ローリングでの最大ドローダウン（上昇トレンド内）。
- 上昇角度の加速: リターンの一次/二次差分。
- イベントスタディ: 過熱シグナル後の継続と反転をそれぞれ OOS で集計し、過熱度で層別。

```python
# 方針（疑似コード）
def detect_fomo(df, ema_n, dev_thr, hh_lb, hh_count_thr):
    ema = df.close.ewm(span=ema_n).mean()
    deviation = (df.close[-1] - ema[-1]) / ema[-1]
    hh_count = count_new_highs(df, hh_lb)
    return deviation >= dev_thr and hh_count >= hh_count_thr
```
