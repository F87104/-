# Long Liquidation — 買い方の投げ

> 急騰で増えた買いが、高値更新の失敗により損切り（売り）を迫られ、
> その売りが下落を加速させる構造を研究する。
> **[Short Squeeze](SHORT_SQUEEZE.md) の完全対称モデル。**

[← SKILL.md に戻る](SKILL.md)

---

## 1. 概要

Long Liquidation（投げ）とは、**買いポジションの損切り（売り）が、下落の燃料になる構造** である。

急騰が起きると「もっと上がる」という続伸期待のもとで新規の買いが積み上がる。
ところが価格が高値を更新できなくなると、買い方は含み損に転じ、損切りの売りを迫られる。
売りは「売り注文」であるため価格は下落し、下落すると更に他の買い方が投げを迫られ、
投げの連鎖（liquidation）が下落を加速させる。

本ファイルは [Short Squeeze](SHORT_SQUEEZE.md) を上下反転させた **完全対称モデル** として記述する。
両者は同じ構造を逆方向から見たものであり、特徴量・検証手法は鏡像対応する。

### Short Squeeze との対称対応表

| 観点 | Short Squeeze（踏み上げ） | Long Liquidation（投げ） |
|---|---|---|
| 起点 | 急落 | 急騰 |
| 期待 | 続落 | 続伸 |
| 失望 | 安値を更新できない | 高値を更新できない |
| 損切り | 売り方の買い戻し | 買い方の投げ売り |
| 加速方向 | 上 | 下 |
| 棚の位置 | 安値圏 | 高値圏 |

## 2. 市場参加者の心理

- **期待していた人**: 急騰に乗って新規買いを入れた参加者。「続伸」を期待。
- **失望の瞬間**: 高値を更新できず、価格が上に伸びない。
- **損切りする人**: 直近で買った参加者。含み損が拡大し、投げ売り（損切り）を迫られる。
- **加速の方向**: **下方向**。投げ売りが売り圧力となり、投げの連鎖を生む。
- **油断していた人**: 「押し目は買い」と機械的に構えていた参加者ほど、連鎖の燃料になる。

> 核心: **上に伸びないこと自体が、下落のエネルギー源になる。**

---

## 3. チャート上の構造

```
急騰
  ↓
買い集中（続伸期待）
  ↓
高値更新失敗（上に伸びない）
  ↓
損切り（買い方の投げ売り）
  ↓
急落（投げの連鎖で下落加速）
```

- 急騰の直後、高値圏で **棚（横ばいの停滞）** が形成されることが多い。
- 棚は「買い方が上を試したが伸びなかった」記録であり、ここに買いの含み損が溜まる。
- 棚を下にブレイクした瞬間、投げ売りが一斉に発生して加速する。

## 4. 数値化できる特徴

| 特徴量 | 観測の狙い |
|---|---|
| **逆V字（山形）** | 急騰幅に対する反落幅の比率。投げの初動を測る |
| **高値圏の棚** | 高値圏での横ばい区間の長さとレンジ幅（ATR比） |
| **棚下抜け** | 棚の下限を終値で割ったか。投げ発火の合図 |
| **Donchian下限更新** | 直近 N バーの安値（Donchian 下限）を割ったか |
| **出来高** | 棚下抜け時の出来高スパイク（投げの集中度） |

---

## 5. 検証仮説

- **H1**: 急騰後に高値圏で棚が形成され、その下限を終値ブレイクすると、
  ブレイク後 M バーで下方向の加速が有意に観測される。
- **H2**: Short Squeeze と Long Liquidation は対称な統計的優位性を持つ
  （ただし市場のロングバイアスにより非対称になる可能性を OOS で検証）。
- **H3**: 高値圏の棚レンジが狭いほど、下抜け後の加速が大きい。

## 6. Python実装候補

- Short Squeeze 検出ロジックの **符号反転版** として実装し、コードを共通化する。
- 高値圏の棚検出、棚下抜け検出、Donchian 下限更新フラグ。
- イベントスタディ: 棚下抜け後のリターン分布を OOS で集計。
  Short Squeeze の結果と並べて **対称性の崩れ** を測定する。

```python
# 方針（疑似コード）— Short Squeeze の鏡像
def detect_long_liquidation(df, rise_lb, shelf_max_atr, donchian_n):
    rise = recent_sharp_rise(df, rise_lb)
    shelf = find_high_range_shelf(df, max_atr=shelf_max_atr)   # 高値圏の横ばい
    breakdown = df.close[-1] < shelf.low
    donchian_break = df.close[-1] < df.low.rolling(donchian_n).min()[-2]
    return rise and shelf.valid and breakdown and donchian_break
```
