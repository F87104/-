# Market Psychology Research Framework

> 市場心理構造研究フレームワーク
>
> ここは「手法集」ではなく **「市場心理構造研究所」** である。
> 勝てる条件を探す場所ではなく、**市場参加者の期待がどこで崩壊したか** を研究する場所である。

**最終更新**: 2026-05-30
**ステータス**: 🔬 研究基盤 (Research Foundation)

---

## 0. このフレームワークの思想

相場は **価格の研究ではない**。
相場とは、**市場参加者の期待・失望・損切り・踏み上げ・投げ売り・FOMO・油断** が
価格という形に投影されたものである。

したがって本研究所では、ローソク足やインジケーターを「予測の道具」としては扱わない。
それらはすべて、次の4つの問いに答えるための **観測装置** である。

```
1. 市場参加者は何を期待していたか？
2. その期待はどこで崩壊したか？
3. 誰が損切りしたか？
4. どちらへ加速したか？
```

価格が動くのは、ある集団の期待が崩壊し、彼らがポジションを投げ、
その投げが次の値動きの燃料になるからである。
本研究所が研究するのは、その **「崩壊 → 投げ → 加速」の構造** である。

## 1. 研究対象とする8つの心理構造

このフレームワークは、市場参加者の心理が価格構造として現れる
代表的な8つのパターンを定義し、それぞれを独立した研究ファイルとして扱う。

|---|---|---|---|

| 2 | **Short Squeeze** | 売り方の踏み上げ | [`SHORT_SQUEEZE.md`](SHORT_SQUEEZE.md) |
| 3 | **Long Liquidation** | 買い方の投げ | [`LONG_LIQUIDATION.md`](LONG_LIQUIDATION.md) |
| 4 | **Trap** | 閉じ込め | [`TRAP.md`](TRAP.md) |

| 7 | **FOMO** | 乗り遅れ恐怖 | [`FOMO.md`](FOMO.md) |
| 8 | **Dormant Breakout** | 長期間放置された節目の更新 | [`DORMANT_BREAKOUT.md`](DORMANT_BREAKOUT.md) |

### 各概念の定義

- **Expectation Failure（期待崩壊）**
  市場参加者が期待している方向へ価格が進まなくなった瞬間。
  「続落するはず」「続伸するはず」という集団的期待が裏切られる地点。すべての構造の起点。

- **Short Squeeze（踏み上げ）**
  急落で増えた新規売りが、価格が下に伸びないことで損切り（買い戻し）を迫られ、
  その買い戻しが上昇を加速させる構造。

- **Long Liquidation（投げ）**
  急騰で増えた買いが、高値更新の失敗により損切り（売り）を迫られ、
  その売りが下落を加速させる構造。Short Squeeze の完全対称モデル。

- **Trap（閉じ込め）**
  ブレイクに飛び乗った参加者が、即座の否定によって含み損を抱え、
  損切りが逆方向への加速を生む構造。ロング/ショート両方向に存在する。

- **Capitulation（降伏）**
  長期間含み損に耐えていた参加者が、最後に一斉に投げ、
  その投げ切りが反転の起点になる構造。

- **Compression（圧縮）**
  低ボラ・停滞の中でポジションが静かに蓄積され、
  均衡が破れた瞬間に一方向へ爆発する構造。

- **FOMO（乗り遅れ恐怖）**
  押し目のない上昇に焦った参加者が追随買いを入れ、
  その追随が更なる上昇を生む自己強化的構造。

- **Dormant Breakout（休眠ブレイク）**
  数ヶ月〜数年触られていない節目が更新され、
  そこに紐づく損益ラインが一斉に変化することで加速が起きる構造。

## 2. 研究方針 — 何を重視し、何を重視しないか

本研究所は、いわゆる「最適化されたトレード手法」を作ることを目的としない。
重視するのは以下である。

### ✅ 重視するもの

| 観点 | 意味 |
|---|---|
| **説明可能性 (Explainability)** | なぜそのシグナルが効くのかを「誰の期待が崩壊したか」で説明できること。ブラックボックスは認めない。 |
| **OOS (Out-of-Sample)** | 学習期間外でも構造が再現するか。インサンプルでのフィッティングは成果と見なさない。 |
| **再現性 (Reproducibility)** | 同じ定義・同じデータで誰が検証しても同じ結論に至ること。 |
| **市場構造 (Market Structure)** | 個別の数値ではなく、参加者の期待と損切りの構造に基づくこと。 |

### ❌ 重視しないもの

- **PF（プロフィットファクター）の最適化** — PF を上げるためのパラメータ探索はしない。
  高 PF は「説明可能な構造」の結果として現れるべきものであり、目的ではない。
- **カーブフィッティング** — 過去に都合よく合わせ込んだ条件は研究成果ではない。
- **指標の組み合わせ探し** — インジケーターの掛け算で勝率を上げる作業は研究ではない。

> **原則**: 「なぜ効くか説明できないエッジは、たとえ過去成績が良くても採用しない。」

## 3. 研究ファイルの共通構造

各心理構造ファイルは、以下の8セクション構造で統一して記述する。

```
1. 概要              … その心理構造の定義と一行サマリ
2. 市場参加者の心理   … 誰が、何を期待し、どこで失望/損切りするか
3. チャート上の構造   … 価格・出来高・ボラがどう現れるか（フロー図）
4. 数値化できる特徴   … 検出のための定量的な観測量の候補
5. 検証仮説          … OOSで検証すべき反証可能な仮説
6. Python実装候補     … 検出・検証のためのスクリプト方針
7. Pine Script実装候補 … TradingView上での可視化・検出方針
8. 今後の研究課題     … 未解決の問い・拡張アイデア
```

この統一構造により、各パターンは「観測 → 数値化 → 仮説 → 検証 → 実装」という
同一のパイプラインに乗せられる。

---

## 4. 研究のワークフロー

```
[観測]   チャート上で「期待の崩壊」を目視で発見する
   ↓
[言語化] 誰が何を期待し、どこで損切りしたかを文章で説明する
   ↓
[数値化] その構造を定量的な観測量に落とす（V字, 棚, ATR拡大 など）
   ↓
[仮説]   反証可能な仮説を立てる（例: 「Xが起きると次のNバーで…」）
   ↓
[Python検証] OOSを含むデータで仮説を統計的に検証する
   ↓
[Pine実装]  再現できた構造のみ TradingView 上で可視化・検出する
   ↓
[研究課題] 残った疑問を「今後の研究課題」に記録し、ループに戻す
```

## 6. 全体の今後の研究課題

- 8つの心理構造の **相互遷移** を定式化する（例: Compression → FOMO → Long Liquidation）。
- 各構造を区別する **共通の特徴量セット** を定義する（V字, 棚, ATR比, 出来高比, EMA乖離, Donchian更新）。
- 「誰が損切りしたか」を推定する **ポジション蓄積モデル** の構築。
- マルチタイムフレームでの構造の入れ子（H4の崩壊がH1でどう見えるか）の研究。
