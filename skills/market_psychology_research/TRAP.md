# Trap — 閉じ込め

> ブレイクに飛び乗った参加者が、即座の否定によって含み損を抱え、
> その損切りが逆方向への加速を生む構造を研究する。

[← SKILL.md に戻る](SKILL.md)

---

## 1. 概要

Trap（閉じ込め）とは、**ブレイクアウトに飛び乗った参加者が「閉じ込められ」、
彼らの損切りが逆方向への燃料になる構造** である。

価格が節目をブレイクすると、ブレイク方向への追随（飛び乗り）が発生する。
ところがブレイクが即座に否定（ダマシ）されると、飛び乗った参加者は含み損を抱える。
彼らは損切りを迫られ、その損切りが逆方向への急走を生む。

Trap は方向によって2種類に分かれる。

- **ロングトラップ (Bull Trap)**: 上抜けブレイクに買いで飛び乗った参加者が閉じ込められ、下落加速。
- **ショートトラップ (Bear Trap)**: 下抜けブレイクに売りで飛び乗った参加者が閉じ込められ、上昇加速。

---

## 2. 市場参加者の心理

- **期待していた人**: ブレイク方向への継続を期待し、飛び乗った参加者。
- **失望の瞬間**: ブレイクが続かず、価格が節目の内側に戻る（ブレイク否定）。
- **損切りする人**: 飛び乗ったばかりの参加者。逃げ場がなく、損切りを迫られる。
- **加速の方向**: ブレイクと **逆方向**。閉じ込められた集団の損切りが燃料になる。
- **油断していた人**: ブレイクを無条件に信頼した参加者。だましの最大の被害者。

> 核心: **ブレイクの否定こそが、逆方向への最大のエネルギーになる。**

### ロングトラップ vs ショートトラップ

| 観点 | ロングトラップ (Bull Trap) | ショートトラップ (Bear Trap) |
|---|---|---|
| ブレイク方向 | 上抜け | 下抜け |
| 飛び乗り | 買い | 売り |
| 否定 | 節目の下に戻る | 節目の上に戻る |
| 損切り | 買いの投げ | 売りの買い戻し |
| 加速方向 | 下 | 上 |

---

## 3. チャート上の構造

```
ブレイク（節目を抜ける）
  ↓
飛び乗り（追随の新規エントリー）
  ↓
否定（ブレイクが続かず節目の内側へ戻る）
  ↓
損切り（飛び乗った参加者の投げ）
  ↓
逆走（ブレイクと逆方向への加速）
```

- 「抜けた → 戻った」という **往復** が Trap の必須要素。
- 否定が速いほど（少ないバー数で戻るほど）閉じ込められた含み損が逃げられず、逆走が強い。

---

## 4. 数値化できる特徴

| 特徴量 | 観測の狙い |
|---|---|
| **ブレイク幅** | 節目をどれだけ超えたか（行き過ぎほど飛び乗りが多い） |
| **否定速度** | ブレイクから節目内側への復帰までのバー数 |
| **ヒゲ比率** | ブレイク方向の長いヒゲ（飛び乗りが拒否された痕跡） |
| **節目内側への終値復帰** | 終値ベースで節目の内側に戻ったか（だまし確定） |
| **出来高** | ブレイク時の出来高と否定時の出来高の比較 |

---

## 5. 検証仮説

- **H1**: 節目を上抜けた後、N バー以内に終値で節目内側へ戻ると（ロングトラップ確定）、
  その後 M バーで下方向の加速が有意に観測される。
- **H2**: 否定が速い（復帰バー数が小さい）ほど、逆走の幅が大きい。
- **H3**: ブレイク方向に長いヒゲ（拒否）を伴うトラップは、ヒゲなしより OOS 再現性が高い。

## 6. Python実装候補

- 節目（直近スイングハイ/ロー、Donchian、ラウンドナンバー）を定義。
- ブレイク → 復帰の往復を検出し、否定までのバー数を記録。
- ロング/ショート両方向を同一関数で符号付き処理。
- イベントスタディ: トラップ確定後のリターン分布を OOS で集計し、否定速度で層別。

```python
# 方針（疑似コード）
def detect_trap(df, level, direction, max_revert_bars):
    broke = broke_level(df, level, direction)              # 節目を抜けた
    reverted, bars = reverted_inside(df, level, direction) # 内側に戻った+所要バー
    return broke and reverted and bars <= max_revert_bars
```

## 7. Pine Script実装候補

- 節目（スイング/Donchian）を描画し、ブレイクと否定の往復を `plotshape` でマーク。
- ロングトラップ・ショートトラップを色分け表示。
- 否定までのバー数を `label` でチャート上に表示。
- `alertcondition` でトラップ確定を通知（観測用）。
