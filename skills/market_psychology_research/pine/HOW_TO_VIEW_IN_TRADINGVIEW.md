# TradingView で検証結果を見る手順

> Python の検証結果を、自分のチャートで再現・確認するためのガイド。

[← SKILL.md に戻る](../SKILL.md)

---

## 1. ファイル

| ファイル | 用途 |
|---|---|
| [`market_psychology_strategy.pine`](market_psychology_strategy.pine) | **Strategy Tester で PF・勝率・トレード数を見る**（検証の確認用） |
| [`market_psychology_visual.pine`](market_psychology_visual.pine) | チャートに「踏み上げ／投げ切り」の印だけ出す（観測用） |

## 2. 手順

1. TradingView でチャートを開き、**時間足を H4（4時間）** にする。
2. 銘柄は **XAUUSD / EURJPY / USDJPY / NAS100 / SPX500** などで試す（**GBPJPY は対象外**）。
3. 画面下「Pine エディタ」に `market_psychology_strategy.pine` を貼り付け → **チャートに追加**。
4. 設定（歯車）→「**検証する構造**」のドロップダウンで **1つだけ選ぶ**:
   - 「① 踏み上げ Short Squeeze (買い)」
   - 「② Capitulation 投げ切り底買い (買い)」 ← 既定はこちら
   ※ ドロップダウン式なので **選んだ方だけ** がエントリーします（両方混ざりません）。
5. 下部の **「ストラテジーテスター」** タブで結果を見る:
   - **Profit Factor（プロフィットファクター）** … 検証の PF と同じ指標
   - **Percent Profitable** … 勝率
   - **Total Closed Trades** … トレード数

> リスクは「1トレード=口座の1%」に固定済み。だから **Profit Factor が R建ての検証PFと直接比較**できます。

## 3. 期待される“目安”（H4・コスト0）

検証(生OHLC)での **銘柄別の傾向**。TradingViewのデータ差・バー数制限で**完全一致はしません**。
見るべきは「**踏み上げ買いがプラス側か**」「投げ継続でないか」という**形**です。

### ① 踏み上げ Short Squeeze（買い）
| 銘柄 | 検証PF（参考） | 件数 |
|---|---|---|
| XAUUSD | 4.06 | 13 |
| EURJPY | 2.55 | 17 |
| AUDJPY | 1.68 | 17 |
| NAS100 | 7.54 | 5 |
| SPX500 | 0.63 | 8 |
| USDJPY | 0.90 | 18 |

→ 全体では FX PF≈1.45 / 指数 1.62。銘柄ごとの件数が少ないので**1銘柄では振れます**。

### ② Capitulation 底買い（買い・日足下降中フィルタON）
| 銘柄 | 検証PF（counter, 参考） |
|---|---|
| NAS100 | 1.86 |
| SPX500 | 1.75 |
| CHFJPY | 1.65 |
| XAUUSD | 1.24（※XAUは上昇押し目の方が良い） |

## 4. 数値が検証とズレる理由（正常です）

- **データソースが違う**（あなたのブローカー/TradingView vs 検証データ F87104）。
- **バー数制限**: 無料プランは過去バーが限られ、古い年が出ないことがある。
- **セッション/H4の区切り**がフィードで異なる。
- **約定の単純化**: 検証は「同じ足で両ヒット＝損切り優先」。TradingViewの埋め方と微差。
- **コスト**: 既定0。実運用はスプレッド/スリッページ分だけ悪化（監査では0.20Rでも踏み上げは PF1.15）。
